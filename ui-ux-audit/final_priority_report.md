# GymTracker UI/UX 종합 우선순위 리포트

> 4개 리포트(UI / UX / 접근성 / 디자인 일관성)를 종합·중복 병합해 우선순위로 재정렬했다.
> 분류 기준: [사용자 영향도 · 수정 난이도 · 핵심목표 관련성 · 반복사용 영향 · 신규이탈 가능성 · 일관성 영향].
> 제품 철칙(헬스장 3초 룰 · 입력 최소화 · 비현실적 목표/하락 단정 금지)과의 관련성을 우선순위에 직접 반영했다.
>
> 캡처 제약: react-native-web 미설치 + 폰 Expo Go 전용 + 인증/온보딩 게이트로 30개 화면 자동 스크린샷 불가 → 4개 리포트 모두 코드 기반 인벤토리로 작성됨. 실제 이미지는 사용자 제공 1장(`screenshots/provided_goalsheet_pullup.png`)뿐.
>
> 출처 약어: UI = ui_audit_report / UX = ux_audit_report / A11Y = accessibility_audit_report / DC = design_consistency_report

P0 근거(한 줄씩)
- 홈 운동시작 부재: 제품 철칙 "3초 판단"의 종착점이 끊긴다. 가장 자주 하는 행동(운동 시작)이 화면 밖에 있어 매 세션 마찰. (반복사용·핵심목표 직격)
- 빨강 위 글자색: WCAG 1.4.3 명백 위반 + 헬스장 빠른 탭 시 검정 on 빨강은 대비 저하. 색·라벨 일괄 치환이라 난이도 최저, 효과 최대(ROI 1위).
- NumPad 위계 역전: 세트 입력은 운동중 가장 빈번한 동작인데 시선이 "다음"으로 끌려가 오인. 색 두 줄 스왑으로 해결.
- 완료 체크 미스탭: 가장 빈번한 단일 동작. 27px+hitSlop 없음 → 장갑·땀 상황에서 짜증 직결. 수치 한 줄 수정.
- 온보딩 초기화 버그: 라벨-동작 불일치로 사용자가 재온보딩 불가. 기능적 버그라 즉시.
- 개발자 도구 노출: 일반 사용자가 데이터를 통째 날릴 위험. 가드 한 줄.

P1 근거(한 줄씩)
- 온보딩 값 편집 부재: 주 4→3회, 증량폭 오선택 등 코어 값을 영구 고정 — 헤비유저 1명 타겟인데 본인 설정을 못 바꾼다.
- 트리거 스펙 역행: "입력 최소화" 철칙 위반. 중급자도 멈칫하는 전문 선택을 신규에 강요(이탈 요인) + default가 스펙과 정반대.
- 흐린 보조 텍스트: 시스템 차원 WCAG 1.4.3 위반. 헬스장 밝은 환경에서 직전기록·휴식·힌트가 안 보임. 토큰 한 곳 상향으로 ROI 최고.
- 종목추가 4탭: 같은 5~8종목 반복하는 헤비유저가 매 운동 4탭 깊이 재탐색 — 반복사용 마찰 누적.
- 아이콘 라벨/벡터: 스크린리더가 글리프를 그대로 읽어 의미 불통(4.1.2) + 이모지가 CARBON 톤과 충돌해 첫인상 신뢰도 저하.
- 노력도 자동 팝업: 다음 세트로 넘어가는 흐름을 모달이 가로챔. 드롭세트 시 타이밍 어긋남 — "추가 입력 0" 철칙 체감 저해.
- 세트행 밀집: 운동중 가장 자주 보는 화면의 스캔성 — 빠른 판단 방해.
- 빨강 의미 충돌: 빨강이 "성공"인지 "위험"인지 학습 부담. 색=의미 1:1 붕괴는 전역 일관성 근원.
- iOS 블루 잔존: 레드 브랜드 전환에서 누락된 4곳 — 화면 내 톤 불일치.
- workout raw hex: 가장 중요한 화면이 토큰 밖이라 홈→운동중 진입 시 회색 톤이 한 단계 밝아짐. 일관성 근본 원인.
- 작은 탭 타깃: WCAG 2.5.5 미달 다발. hitSlop 일괄 추가는 저비용.
- 리포트 CTA 한 단계: 행동 연결이 가장 잘 된 화면인데 마지막 한 탭이 끊김 — 작은 수정으로 완결.
