> 본 회차는 추측 없이 "이미지에서 실제로 본 것"만 적는다. 캡처는 web 렌더라 네이티브와 폰트 힌팅·터치영역이 약간 다를 수 있음(터치타깃 px는 코드 수치 신뢰).

## 캡처 전반에서 먼저 보인 것 (전 화면 공통)

1. **테스트 계정이 "이번 주 데이터 0" 상태다.** 홈 출석 0/5, 모든 부위 0/12세트, 리포트 출석 0%로 비어 있다. 코드 감사가 "정보 우선순위 우수"로 칭찬한 블록2(이번 주)·리포트 출석 링이 **실데이터가 비면 거대한 빈 공간**으로 보인다는 점은 코드만으론 못 본 부분. (아래 화면별 P1)
2. **레드 액센트는 일관되게 잘 깔려 있다.** 탭바 활성 빨강, 펠릿/세그먼트 ON 빨강, CTA 빨강 — 이미지상 톤은 통일돼 보인다. 코드 감사가 우려한 "iOS 블루 잔존"은 본 8장 캡처에는 안 드러남(운동중 세그먼트 미캡처).
3. **이모지 아이콘 문제는 이미지상 생각보다 약하다.** 홈 우상단 기어(⚙️)는 보이지만, 코드 감사가 강조한 💬 챗 FAB는 작고 화면 우하단에 묻혀 거의 안 보인다. 로그인 💪·"G"는 본 캡처에 없어 재검증 불가.

## 1. 홈 `web-home.png` — 3초 룰 집중 검증

**코드감사 검증 결과**
- [맞음] "오늘 확인 필요" 빈 상태 카피("지금 따로 챙길 종목은 없어요. 평소대로 기록만 쌓으면 돼요.") 실제로 안심형·중립 — 칭찬 정당.
- [맞음] greet "월요일 · 점진적으로 강해지는 중"이 13px muted라 **실제로 흐리고 작다**. 이미지에서 거의 배경에 묻힌다. 헤더 타이틀 부재 지적도 시각 확인됨(좌상단 비고 큰 제목 없음, 기어만).
- [맞음] "운동 시작" 버튼이 홈 본문에 **실제로 없다**. CTA로 손 갈 곳이 하단 탭바 빨강 play뿐 — P0 지적 시각 확인.
- [과장 아님/오히려 강화] "확인 필요/이번 주/핵심 종목" 3블록 위계는 좋으나, **현재 캡처에선 블록2(이번 주)가 화면 세로의 절반 가까이를 먹는다.** 0/5·0/12 빈 막대 5줄 + "부족한 부위 …1세트" 텍스트가 길게 깔려, 정작 행동 블록(확인 필요)과 핵심 종목이 한 화면에서 밀려난다.

**이미지에서 새로 발견**
- (P1) **빈 상태 막대 5줄이 시각 노이즈다.** 가슴/하체/등/어깨/코어가 전부 0세트라 회색 빈 트랙만 5개 반복 — "잘 가고 있나"를 못 주고 오히려 "아무것도 안 했다"는 부정 인상. 빈 주에는 막대를 접고 "이번 주 시작 전" 한 줄이 나아 보인다. (`index.tsx:116` volBlock)
- (P1) **핵심 종목 카드 배지 "반복수 쌓는 중"이 빨강 테두리 칩이라 시선을 강하게 끈다.** 정작 중요한 "오늘 총 4회"보다 빨강 배지가 먼저 보인다 — 위계상 배지가 과해 보임. 두 번째 카드(Cable Crunch)는 배지가 카드 밖으로 잘려("반복수 쌓는…") 시작된다(스크롤 경계).
- (P2) 홈 진입 화면에서 핵심 종목은 첫 카드 1.5개만 보인다 — 3초 안에 "오늘 뭐부터"가 스크롤 아래 숨음. 블록2 압축이 곧 3초 룰 회복.

**우선순위: P0**(운동 시작 부재) + P1(빈 주 블록2 과점유·배지 위계)

## 2. 운동 `web-workout.png` — ⚠ 캡처가 운동중 화면이 아님

**중요: 이 캡처는 "운동 중 세트 입력" 화면이 아니라 운동 "시작" 화면(지난 운동 불러오기 리스트)이다.** 코드 감사의 핵심 타깃이던 목표 배너·세트행·완료 체크·NumPad는 **이 이미지에 없다** → 운동중 시각 재검증 불가(별도 캡처 필요).

**그래도 이 시작 화면에서 본 것**
- [새 발견 P1] **세션 카드마다 좌측에 빨강 세로 바가 붙어 있다.** 의미 불명(전부 같은 빨강)이라 장식인지 상태인지 모호 — 빨강을 의미 없이 또 쓰는 사례. (`components/SessionCard.tsx`)
- [새 발견 P1] **"이대로 시작" 버튼이 빨강인데, 카드마다 1개씩 반복**돼 화면에 빨강 버튼이 4개 쌓인다. 어느 걸 골라야 할지 위계 없이 동등 — 헤비유저도 한 박자 스캔. 상단 부위 펠릿(전체/등/코어…)과 합쳐 컨트롤이 많다.
- [맞음, UX리포트] 시작 옵션이 "지난 운동" 리스트로 떨어진다는 UX 지적(리포트 CTA·홈 시작 동선) 시각 확인 — 운동 시작이 곧 "과거 복사 고르기"가 된다.
- [맞음] "7종목·27세트·1시간48분" 메타가 빨강 숫자라 가독 좋음. 날짜·부위 배지 위계도 깔끔.

**우선순위: P1** + **운동중 화면 재캡처 필요(P0 항목 재검증 미완)**

## 5. 리포트 `web-reports.png` — 다음 행동 연결 집중

**코드감사 검증 결과**
- [맞음, 강하게] "다음 주 액션" 히어로 + "오늘 운동 시작하기 ›" 빨강 CTA가 **최상단에 크게** — 행동 연결 모범, 시각 확인. 앱에서 위계가 가장 잘 잡힌 화면 맞음.
- [맞음] 액션 번호 원(①) + 텍스트 구조 깔끔.

**이미지에서 새로 발견 (코드만으론 못 본 것)**
- (P1) **"핵심 종목 변화" 카드가 줄글 6줄로 빽빽하다.** "Incline Bench Press 50kg×8회·1세트 → 50kg 총 9~10회" 식 문장이 6종목 연속 — 헬스장 3초 룰과 정반대로 **읽어야 하는 문단**이다. 화살표(→) 전후 숫자가 한 줄에 다 들어가 어디가 현재/목표인지 스캔 어려움. 표나 칩으로 분해 필요. (`components/report/ReportScreen.tsx`)
- (P1) **"이번 주 출석 0%" 도넛이 크게 비어 있다.** 빈 주엔 0% 큰 링이 부정 신호로 화면 중앙을 점유 — 리포트 첫인상이 "0%"가 됨. "진행 중" 배지(주황)는 좋으나 0% 자체가 너무 크다.
- (P2) "다음 주 액션"이지만 진입은 "이번 주 리포트" 헤더 — 시점 라벨 혼동(UX 지적 시각 확인). CTA "오늘 운동 시작하기"와 "다음 주 액션"의 시제가 한 카드에 섞임.
- [과장 아님] 액션 텍스트 "핵심 종목 총 반복수를 1~2회씩 늘려보기"는 한 줄로 명확 — 좋음.

**우선순위: P1**(핵심종목 변화 줄글 밀집 + 0% 링 과점유)

- **A1 (P1) 빈 주 상태의 시각 붕괴 — 전역.** 이번 주 데이터 0이면 홈 블록2(빈 막대 5줄)·리포트 출석 도넛(0% 대형)이 거대 빈 공간/부정 신호로 화면을 점유한다. 빈 주 전용 축약 상태 필요. 코드 감사는 이 상태를 못 봄(데이터 있다고 가정). `index.tsx:116`, `ReportScreen.tsx`.
- **A2 (P1) 리포트 "핵심 종목 변화"가 줄글 6줄 문단.** "A 50kg×8 → 50kg 총 9~10회" 식 문장 6개 연속이라 3초 룰 위반 — 표/칩으로 분해. `components/report/ReportScreen.tsx`.
- **A3 (P1) 목표설정·온보딩 화면의 거대 빈 공간.** goals는 화면 60% 공백, 온보딩은 상단 1/3 공백. goals에 온보딩 코어값 편집을 넣으면 빈 공간 + UX편집부재(기존 P1) 동시 해결.
- **A4 (P1) 캘린더 이상치 노출.** "4시간 25분" 같은 비정상 세션 시간이 그대로 표시됨 — 표시 상한/이상치 처리.
- **A5 (P1) 운동 시작 화면 빨강 과다.** 세션 카드 좌측 의미 불명 빨강 세로 바 + 카드마다 "이대로 시작" 빨강 버튼 반복 → 빨강 4개 동등 위계. `components/SessionCard.tsx`.
- **A6 (P2) 홈 핵심종목 배지 위계 역전.** "반복수 쌓는 중" 빨강 테두리 칩이 정작 중요한 "오늘 총 4회"보다 강하게 보임 + 두 번째 카드 배지 스크롤 경계서 잘림.
- **A7 (P2) 종목허브 펠릿 자물쇠 이모지(🔒) 의미 불명** + 부위 도트 색 범례 없음.
- **A8 (P2) 캘린더 날짜 전부 빨강 굵게 → 강조 인플레.** 7개 연속 빨강 날짜로 위계 평평.

### 수정 (코드 감사 과장/오판 — 이미지상 실제론 약함)
- **M1 캘린더 restText "거의 비가시"는 과장.** 회색이지만 읽힘. 색 상향은 "선택" 수준(P2 유지하되 심각도 하향).
- **M2 종목허브 활성 펠릿 "선택 거의 안 보임"은 과장.** 이미지상 더 밝은 회색으로 충분히 구분됨. ACCENT_TINT는 개선이지 결함 아님.
