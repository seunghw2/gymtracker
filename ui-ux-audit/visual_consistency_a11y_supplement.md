# GymTracker 시각 일관성·접근성 보강 감사 (실제 캡처 교차 비교)

근거: `ui-ux-audit/screenshots/test/`의 실제 캡처 18장을 나란히 비교한 결과만 기록한다. 추측 없음. 기존 코드 기반 감사(`design_consistency_report.md`, `accessibility_audit_report.md`)를 이미지로 재검증·보강한다.

본 화면들(web-* 캡처)에서 코드 감사가 지목한 "운동 중 화면의 iOS 그레이(#1C1C1E/#2C2C2E)" 계열은 직접 노출되지 않았다(실제 세트 입력 중 화면은 캡처에 없음). 대신 진입 화면들에서만 검증 가능했다. 그 한계 안에서 본 것만 적는다.

## A. 화면 비교로만 드러난 새 발견 (코드 감사에 없거나 약하게 다룬 것)

### A-1. 같은 의미의 메인 CTA가 화면군마다 빨강 vs 초록으로 갈린다 [심각도: 높음] ★최우선
가장 눈에 띄는 시각 모순. "운동 시작" 계열 액션이:
- **web-templates.png**: "이 루틴으로 시작" 버튼 2개가 **완전한 초록**(밝은 라임그린), 카드 제목·"추천" 뱃지·부제까지 전부 초록 톤. 화면 전체가 초록 테마.
- **web-reports.png / web-ai-reports.png**: 같은 진입 액션 "오늘 운동 시작하기 ›"는 **빨강**.
- **web-home.png**: 메인 시작은 탭바 중앙 **빨강** 원형 FAB.
- **web-ai-index.png**: "운동하러 가기"도 **빨강**.

즉 사용자가 리포트/홈에서는 빨강으로 운동을 시작하다가, 템플릿 화면에 오면 똑같은 "시작" 동작이 초록으로 바뀐다. 코드 감사는 "workout의 iOS 블루 잔재"를 지적했지만, **이미지에서 실제로 튀는 건 블루가 아니라 템플릿의 초록 전면**이다. 초록은 앱 어디서도 의미색(증가/READY)으로 쓰는 색인데(web-exercises의 ↑5kg 초록, 홈 "반복수 쌓는 중" 등), 템플릿에서는 그게 CTA 배경색으로 쓰여 의미가 충돌한다.
→ 통일: 템플릿 CTA를 ACCENT(빨강)로. 초록은 "증가/달성" 의미색으로만 한정.

### A-2. 빨강 CTA 위 글자색은 이미지상 전부 흰색으로 보인다 — 코드 지적과 불일치 [심각도: 중, 코드 감사 과장 가능]
코드 감사 #1은 "빨강 버튼 위 글자가 어떤 화면은 검정"이라 강하게 지적했다. 그러나 **실제 캡처에서 본 모든 빨강 버튼의 글자는 흰색**이다:
- web-reports "오늘 운동 시작하기 ›" → 흰 글자
- web-goals "저장" → 흰 글자
- web-trainer "확인" → 흰 글자
- web-ai-index "운동하러 가기" → 흰 글자
- web-body-parts "추가" → 흰 글자
검정 글자 빨강 버튼은 이 18장 어디에도 안 보인다. 코드의 `#000000` 라벨은 **세트 입력 중(finish/save) 화면 한정**일 가능성이 높고, 그 화면은 캡처에 없다. → 코드 감사의 "전반적 위반"은 진입 화면 기준으로는 과장. 단, workout 내부 캡처가 없어 완전 반증은 못 함. "운동 중 화면만 확인 필요"로 좁혀야 한다.

### A-3. 두 가지 헤더 패턴이 실제로 공존 — 이미지로 확정 [심각도: 중, 코드 감사 #4 확인]
캡처가 코드 #4를 그대로 입증한다:
- **좌측 큰 제목(약 28~30)**: web-home(없음, 인사말만), web-exercises("종목"), web-calendar("기록"), web-settings("설정") — 좌측 상단 큰 볼드.
- **중앙 네비 제목(약 17)**: web-reports("이번 주 리포트"), web-goals("목표 설정"), web-account("계정"), web-body-parts("부위 관리"), web-custom-exercises("커스텀 운동"), web-workout/templates/exercise-add, web-trainer.

특히 **리포트는 탭 루트인데도 중앙 작은 제목**이라, 옆 탭(종목·기록은 좌측 큰 제목)과 헤더 격이 달라 보인다. 탭바를 오가면 제목 위치·크기가 좌상단↔중앙으로 점프한다. 이미지로 확실히 보인다.

### A-4. 뒤로가기 표기가 화면마다 다르다 [심각도: 중, 신규]
푸시된 화면의 좌상단 back affordance가 제각각:
- web-goals / web-account / web-body-parts / web-custom-exercises: **"‹ 설정"** (셰브론+텍스트 라벨, 빨강)
- web-templates / web-trainer / web-ai-* / web-workout: **"‹"만** (텍스트 라벨 없는 빨강 셰브론, 단독)
- web-exercise-add: **"← 뒤로"** (화살표+"뒤로" 텍스트)
세 가지 뒤로가기 표기(라벨 있는 셰브론 / 셰브론 단독 / 화살표+뒤로)가 공존한다. 코드 감사는 셰브론 크기/색만 지적했으나, 이미지에서는 **표기 형태 자체가 3종**으로 더 큰 불일치다.

### A-5. AI 화면군이 별도 비주얼 언어를 쓴다 [심각도: 중, 신규]
web-chat / web-trainer / web-ai-intake / web-ai-index는 다른 화면과 톤이 다르다:
- **이모지 아바타 전면 사용**: 트레이너 캐릭터가 컬러 이모지(👼🌸 등)로, 카본(흑+적) 톤과 충돌. web-chat 상단 아바타도 컬러풀.
- **빨강 외곽선 알약 칩**(web-chat "가슴/하체/등…", web-ai-intake 선택지)이 다른 화면의 채움 칩(web-calendar 부위 태그=회색 채움, web-exercises 그룹칩=회색 채움)과 스타일이 다르다. 같은 "선택 칩"인데 AI는 빨강 아웃라인, 나머지는 회색 솔리드.
- web-ai-intake 선택지 버튼은 **이모지+텍스트** 라벨이라 다른 화면 버튼(텍스트만)과 결이 다름.

### A-6. 빈 상태 메시지 패턴이 화면마다 다르다 — 이미지로 확정 [심각도: 중, 코드 감사 #6 확인+보강]
- web-ai-index: **📬 이모지 + 굵은 제목 + 설명 + 빨강 CTA** (가장 완성형)
- web-templates: **이모지 없이 회색 2줄 텍스트만**("저장된 템플릿이 없어요…"), CTA 없음
- web-home "오늘 확인 필요": **회색 박스 안 안내문**("지금 따로 챙길 종목은 없어요…")
세 빈 상태가 완성도·구조가 모두 다르다. ai-index만 잘 만들었고 나머지는 빈약. 코드 #6 지적이 이미지로 확인됨.

## B. 접근성 — 이미지로 재검증

### B-1. 흐린 회색 텍스트는 실제로 읽기 어렵다 — 코드 감사 맞음 [심각도: 높음, 확인]
캡처에서 육안으로 확인되는 저대비:
- **web-home**: "월요일 · 점진적으로 강해지는 중" 인사말, "부족한 부위… 1세트" 캡션, 출석 도트 미점등 회색 — 검은 배경에서 흐릿하게 묻힌다.
- **web-home/exercises**: 종목 부제("핸몸", "등 · Bodyweight", "머신 / 케이블"), "지난 기록…" 줄이 매우 어둡다.
- **web-reports**: "이번 주 점진적 과부하 단계" 부제가 제목 바로 아래에서 거의 안 보임.
- **web-account**: "모든 운동 기록이 삭제되며 되돌릴 수 없어요" 경고문이 흐린 회색 — **삭제 경고인데 가독성이 가장 낮은 축**. 위험 안내가 흐린 건 문제.
- **web-templates**: 빈 상태 2줄, **web-onboarding**: "목표에 맞춰 과부하 방식을…" 안내문도 흐림.
코드 감사의 "muted/#48484A/#555 남용" 지적은 이미지로 확실히 뒷받침된다. 특히 web-account 경고문과 web-onboarding 안내문이 시각적으로 가장 약하다.

### B-2. 색만으로 상태 구분 — 이미지상 실제로 구분 어려운 곳 [심각도: 중, 부분 확인]
- **web-home 출석 도트**(● ● ● ● ●): 0/5라 전부 회색이라 비교 불가했지만, 점등=빨강/미점등=어두운 회색 설계상 색 의존. 다만 옆에 "0/5회" 텍스트가 있어 보조됨(코드 감사대로 OK).
- **web-reports 진행바**("목표 부위 주간 세트"): 가슴/하체 막대가 **어두운 초록**인데 거의 빈 상태라 0%와 구분이 약하다. 옆 "0 부족" 텍스트가 보조. 진행이 차오를 때 색만으로는 약할 것.
- **web-home 단계 뱃지**("반복수 쌓는 중" 빨강 아웃라인): 텍스트 라벨이 있어 색맹에도 OK. 코드 감사대로 라벨 동반은 통과.
→ 코드 감사 결론(텍스트 라벨 동반된 곳은 OK, 진행바·도트는 보강 권장)이 이미지로도 타당.

### B-3. 작은 터치 타깃 — 이미지로 확인되는 곳 [심각도: 중~높음, 확인]
- **web-home 우상단 ⚙️**: 작은 단독 아이콘, 라벨 없음.
- **web-workout / templates의 "···" 메뉴**, web-workout 카드 우상단 작은 점 3개: 작아 보임.
- **web-exercises "× 그룹"**: 잠금 자물쇠 칩들 옆 작은 요소.
- **web-calendar 타임라인엔 작은 점/네비 없음**(이 캡처는 큰 타깃 위주).
코드 감사의 "28px 이하 + hitSlop 미지정" 다수 지적은 직접 px 측정은 못 하나, ⚙️·····× 같은 작은 단독 아이콘이 이미지에서도 작게 보여 방향성은 맞다.

### B-4. 이모지 vs 벡터 혼재 — 이미지로 확정 [심각도: 중, 코드 감사 #8 확인+보강]
- **web-settings**: 행 아이콘이 **깔끔한 벡터**(목표설정·휴식시간·부위관리·커스텀운동·알림·달력) — 단일 톤, 카본과 잘 맞음.
- **web-home 우상단 ⚙️**: 같은 "설정" 진입인데 **이모지 기어** — settings 탭 벡터와 불일치(코드 #8 정확).
- **web-account 로그아웃**: 벡터형 아이콘.
- **AI 화면 전체**: 이모지 아바타(👼🌸 등) + web-ai-intake 선택지 이모지(💪🏆🔥📅❤️🤔) + web-ai-index 📬.
→ 설정 탭만 벡터로 잘 통일됐고, 홈 기어·AI군은 이모지. 코드 #8이 이미지로 정확히 입증됨. settings의 벡터 세트를 기준으로 삼으면 된다.

## C. "코드 감사 대비 추가/수정" 리스트

**추가(이미지로만 보인 신규 발견):**
1. **[추가·최우선]** 템플릿 화면의 초록 CTA 전면(web-templates) — 같은 "시작" 액션이 빨강↔초록으로 갈림. 코드 감사가 놓친 가장 큰 시각 모순.
2. **[추가]** 뒤로가기 표기 3종("‹ 설정" / "‹" 단독 / "← 뒤로") 공존(A-4).
3. **[추가]** AI 화면군의 별도 비주얼 언어 — 컬러 이모지 아바타 + 빨강 아웃라인 칩(다른 화면은 회색 솔리드 칩)(A-5).
4. **[추가]** web-account 삭제 경고문이 흐린 회색 — 위험 안내인데 가독성 최하(B-1).

**수정(코드 감사가 과장/부정확):**
5. **[수정]** 코드 #1 "빨강 버튼 위 검정 글자 전반 위반" → 진입 화면 캡처에선 전부 흰 글자. 검정 라벨은 **운동 중 세트 화면 한정**으로 범위 축소 필요(캡처 없어 미확정, "확인 대상"으로).
6. **[수정]** 코드 #5 "iOS 블루(#0A84FF) 잔재가 튄다" → 진입 화면 캡처엔 블루가 안 보임. 실제로 튀는 건 블루가 아니라 **템플릿 초록**. 블루는 세트 화면 한정으로 보임.

**확인(코드 감사가 이미지로 입증됨):**
7. 헤더 두 패턴 공존(#4) — 확정, 특히 리포트 탭이 중앙 작은 제목으로 이질적.
8. 흐린 회색 텍스트 남용(접근성 #2,#3) — web-home/account/onboarding/reports에서 육안 확인.
9. 이모지 vs 벡터 혼재(#8) — settings 벡터 vs 홈 기어/AI 이모지로 확정.
10. 빈 상태 패턴 불일치(#6) — ai-index(완성형) vs templates(빈약)로 확정.
11. 색만 구분은 텍스트 라벨로 대체로 보조됨 — 진행바만 보강 권장(접근성 #8과 일치).

## D. 우선순위 (이미지 근거 기준)

| 순위 | 항목 | 근거 캡처 | 비고 |
|---|---|---|---|
| P0 | 템플릿 초록 CTA → 빨강 통일(같은 "시작" 의미색 일원화) | web-templates vs web-reports/home | 코드 감사에 없던 최대 모순 |
| P0 | 흐린 회색 정보·경고 텍스트 상향(특히 삭제 경고) | web-account, onboarding, home | 접근성 직접 영향 |
| P1 | 헤더 패턴 통일(리포트 탭 좌측 큰 제목으로) + 뒤로가기 표기 1종 | reports vs exercises/calendar; A-4 | |
| P1 | 홈 ⚙️ 등 이모지 → settings 벡터 세트로 | web-home vs web-settings | |
| P2 | AI 화면 칩/아바타 톤을 본 앱과 정렬(빨강 아웃라인 칩 vs 회색 솔리드) | web-chat, ai-intake | |
| P2 | 빈 상태 ai-index 패턴을 templates 등에 확산 | ai-index vs templates | |
| -- | 운동 중(세트 입력) 화면 캡처 확보 후 빨강 위 검정 글자·블루 잔재 재확인 | (캡처 없음) | 코드 #1,#5 미확정분 |

근본 처방은 코드 감사와 동일하다: **토큰을 실제로 쓰게 하기.** 단 이미지로 보면 1순위 교정 대상은 "빨강 위 검정 글자"가 아니라 **템플릿 초록 CTA와 흐린 경고문**이다.
