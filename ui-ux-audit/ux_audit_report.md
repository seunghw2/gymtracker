## 0. 평가 전 발견한 구조적 마찰 (모든 시나리오에 영향)

### A. 목표 시스템이 두 갈래로 단절됨 — High
- 온보딩(`app/onboarding.tsx:88` `handleNext` → `saveGoalSetting`)은 **점진과부하 엔진 입력**(goalType / weeklyFrequency / incUpper / incLower / progressionTrigger)을 `useOverloadStore`에 저장한다.
- 설정의 "목표 설정"(`app/(tabs)/settings.tsx:83` → `app/goals.tsx`)은 **완전히 다른 값**(goalWeightKg / goalBodyFatPct / restDurationSec / unitKg)을 `getSetting/setSetting`으로 저장한다.
- 결과: 온보딩에서 정한 "목적·주간 횟수·증량폭"을 **온보딩 이후 다시 수정할 화면이 없다.** 주 4회 → 3회로 바뀌어도, 증량폭을 잘못 골랐어도 손댈 길이 없다. 코어 제품 값인데 편집 불가.

### B. "온보딩 초기화"가 다른 온보딩을 지움 — High (버그성)
- 설정의 `개발자 도구 (온보딩 초기화)`(`settings.tsx:116` → `handleResetOnboarding` → `deleteAiProfile`)는 **AI 인테이크 프로필**만 지운다.
- 홈 게이트(`app/(tabs)/index.tsx:46` `if (!gs?.onboarded) router.replace('/onboarding')`)를 결정하는 `goalSetting.onboarded`는 건드리지 않는다.
- 사용자가 "온보딩을 다시 하겠다"고 눌러도 5단계 온보딩은 다시 안 뜬다. 라벨과 동작 불일치.

### C. 홈에 "운동 시작" 진입점이 없음 — High
- `app/(tabs)/index.tsx` 전체에 workout/운동시작 CTA가 **0개**(grep 결과 없음). 운동 시작은 오직 탭바 가운데 play 버튼(`components/CustomTabBar.tsx:92`)뿐.
- 홈은 "오늘 확인 필요 / 이번 주 / 종목 리스트"만 보여주고 정작 행동(운동 시작)으로 이어주는 버튼이 없다. 3초 판단 후 손이 갈 곳이 화면 밖(탭바)에 있다.

### D. 스펙 대비 구현 드리프트(혼란 유발 잔재) — Med
- 온보딩이 스펙(3개 질문·트리거 two_sessions 고정)과 달리 **5단계 + 증량 트리거 선택 UI**(`onboarding.tsx:30` `TRIGGER_META`, default `'single'`)가 살아 있음. 스펙 §5/§18은 트리거를 묻지 말라 했는데 묻는다.
- 종목 상세(`app/exercise/[name].tsx`)는 스펙 §13/§18의 "축소판(1RM 1차트 + 최근 기록 표)"과 달리 **5지표 토글(1RM/최대무게/볼륨/빈도/렙) + 빈도 히트맵 + LLM 코치라인 + AI 구조화 리포트 + 편집 체크리스트**가 전부 남아 있음(`[name].tsx:27-31, 108-114`).
- 이 잔재들은 "MVP 제외"로 선언됐지만 화면엔 그대로라 사용자에게 미완성·과부하 인상을 준다.

## 시나리오 1 — 첫 가입 → 온보딩 → 목표 설정 → 홈

### 현재 흐름
1. 미로그인 → 인증 게이트가 `/(auth)/login`으로 강제(`app/_layout.tsx`). 로그인 화면은 Apple/Google/Kakao 버튼만(`app/(auth)/login.tsx:97-133`). 깔끔함.
2. 로그인 성공 → `/(tabs)` → 홈 진입 즉시 `!onboarded`면 `/onboarding`으로 replace(`index.tsx:46`).
3. 온보딩 5단계(`onboarding.tsx`): ① 목표 4지선다 ② 추적 종목 선택(`exercise-add` 모달 왕복) ③ 주간 빈도 stepper ④ 종목별 규칙 + **증량 판정 시점 3지선다** ⑤ 상/하체 증량폭 stepper → `시작하기`.
4. `handleNext`(`onboarding.tsx:88`)가 `saveGoalSetting` + `bulkCreateGoals` 후 `/(tabs)`로 replace.

### 문제점
- **4단계가 사용자에게 결정을 떠넘긴다.** "증량 판정 시점(한 세션 / 2세션 / +RPE)"은 중급자도 "뭐가 맞지?" 멈칫하는 전문 선택이다. 스펙은 이걸 `two_sessions`로 고정하고 안 묻기로 했는데(§5), 화면엔 default가 오히려 `'single'`(`onboarding.tsx:54`)로 잡혀 스펙과 정반대. "입력 최소화" 철칙 위반.
- **온보딩 종목 선택이 모달을 떠나 `exercise-add`로 라우팅**(`onboarding.tsx:61` `openPicker` → `router.navigate('/exercise-add')`)된다. 온보딩 도중 다른 스택으로 나갔다 콜백(`setExercisePickCb`)으로 돌아오는 구조라, 첫 사용자에겐 "온보딩이 끊겼나?" 하는 맥락 단절이 생긴다.
- **온보딩 직후 홈이 비어 보인다.** 막 종목만 골랐을 뿐 기록이 없어 블록1 "오늘 확인 필요"는 전부 `NEED_BASELINE`(기준 만들기)로 채워지고, 블록2 출석은 0/4, 부위 볼륨 없음. 첫인상이 "할 게 없는데 시작 버튼도 안 보이는" 화면(시나리오3·발견 C 연동).
- **온보딩을 잘못 채워도 되돌릴 데가 없다**(발견 A). 5단계를 빠르게 넘긴 사람일수록 나중에 후회하는데 수정 경로가 막힘.

### 개선안
- 4단계의 트리거 선택 UI 제거하고 `two_sessions` 고정(스펙대로). 정 남기려면 "고급" 접힘 안으로. → 온보딩 5단계 → 4단계로.
- 종목 선택을 온보딩 내부 인라인 시트로(현재처럼 풀스택 라우팅 말고). 최소한 돌아왔을 때 진행바·단계가 유지됨을 시각적으로 보장.
- 온보딩 마지막 단계 또는 직후 홈 상단에 "첫 운동 시작하기" 1버튼을 노출해 baseline 만들기로 바로 유도.
- 설정 "목표 설정"(`goals.tsx`)에 온보딩 값(목적·주간횟수·증량폭) 편집 섹션을 합치거나, 온보딩 재진입 경로를 진짜로 연결.

### 우선순위: High (A·발견 + 트리거 스펙역행)

## 시나리오 2 — 운동 시작 → 종목 추가 → 세트 입력 → 노력도 → 종료

### 현재 흐름
1. 탭바 play(`CustomTabBar.tsx:92`) → 바텀시트 3선택: 템플릿 / 기존 기록 복사 / 빈 운동(`CustomTabBar.tsx:109-131`).
2. 빈 운동 → `createWorkoutSession` 후 `/workout`(`handleStartEmpty:67`). 종목 추가는 `openExerciseSelect`(`workout.tsx:782`) → 부위→장비→(브랜드)→목록 다단 모달 또는 검색.
3. 세트 입력: 세트행 무게/횟수 탭 → 앱 자체 NumPad(`components/NumPad.tsx`)로 입력, ✓로 완료(`handleCompleteSet:879`). 완료 시 휴식 타이머 자동 시작 + 직전값 기준 다음 세트 라벨.
4. 추적(core/support) 종목엔 1줄 목표 배너(`workout.tsx:1721-1754` "오늘 {todayTarget} / 남은 N회"), 탭하면 단계·비교·성공조건 시트.
5. 핵심 종목 마지막 작업 세트 완료 → 0.8초 뒤 노력도 4버튼 시트(`workout.tsx:930-942, 2243`). 스킵 가능.
6. 종료: 부위 미선택이면 부위 선택 먼저(`handleFinishWorkout:496`) → 완료 확인 Alert → 읽기전용 요약(볼륨/세트/종목/PR + 세션평가).

### 문제점
- **시작 옵션이 스펙(2개)보다 많고(3개) 가운데 두 옵션 라벨이 헷갈린다.** "템플릿으로 시작" vs "기존 운동 기록으로 시작"은 헤비유저도 한 박자 읽어야 구분된다. 매번 운동마다 거치는 관문이라 누적 마찰.
- **종목 추가가 다단계 드릴다운**(부위→장비→브랜드→목록). 검색이 있긴 하나(`modalSearch`), 헤비유저는 같은 5~8종목을 반복하는데 매번 4탭 깊이를 들어간다. 최근/즐겨찾기(`favIds`)가 있으나 기본 진입이 "부위 선택"이라 빠른 경로가 한 단계 가려져 있다.
- **NumPad의 "다음" 버튼이 검정 글씨/빨강 배경**(`NumPad.tsx:108-110` `nextBtn backgroundColor #FF3B30`, `nextText color #000`)으로, 빨강+검정은 경고/삭제처럼 읽혀 "다음 칸 이동"이라는 안전 동작과 시각 의미가 충돌. 손은 바쁘고 눈은 0.5초인 맥락에서 오인 위험.
- **목표 배너의 "남은 N회"가 대표 무게 세트만 합산**(`workout.tsx:1730-1734`)한다. 워밍업 후 무게를 바꾸거나 드롭세트가 끼면 doneTopReps 계산 기준이 직관과 어긋나 "왜 안 줄지?" 혼란 가능. 단정적 숫자라 신뢰도 낮을 때 더 위험.
- **노력도 시트가 0.8초 지연 후 자동 팝업**(`setTimeout(...,800)`). 사용자가 바로 다음 세트/종목으로 넘어가려는 순간 모달이 가로채 흐름을 끊는다. 마지막 세트 판정이 살짝만 어긋나도(드롭세트 등) 엉뚱한 타이밍에 뜬다.
- **종료가 Alert 연쇄.** 부위 선택 → "운동 완료?" Alert → 완료 → 요약 모달. 시스템 Alert이 2번 끼어 "추가 입력 0" 철칙과 체감이 다르다.

### 개선안
- 시작 시트를 "최근 루틴(템플릿+지난운동 합친 카드 리스트) / 빈 운동" 2갈래로 단순화. 라벨 텍스트 재작성("템플릿/기존 기록" 용어 통일).
- 종목 추가 모달 첫 화면을 "최근·즐겨찾기" 우선으로(부위 드릴다운은 보조 탭). 헤비유저 반복 종목 1~2탭 도달.
- NumPad "다음"을 중립색(파랑/회색)으로, "완료"만 강조색. 빨강은 삭제/위험에만.
- 노력도 시트 자동 지연 팝업 대신, 마지막 세트 행에 인라인 노력도 칩을 띄우거나 종료 요약에 모아 받기. 최소한 지연 800ms 제거하고 즉시/명시적으로.
- 종료 확인 Alert을 요약 모달의 "종료" 버튼으로 흡수(이중 확인 제거).

### 우선순위: High (종목 추가 깊이·노력도 타이밍), Med (NumPad 색·시작 라벨)

## 시나리오 3 — "오늘 뭘 할지" 홈에서 3초 판단

### 현재 흐름
1. 홈(`app/(tabs)/index.tsx`) 3블록: ① 오늘 확인 필요(READY_TO_INCREASE/STALL/NEED_BASELINE) ② 이번 주(출석 도트 + 부위 볼륨 바 + 부족 부위) ③ 종목 리스트(핵심 펼침 / 보조·기록만 접힘).
2. 행 탭 → `ExerciseGoalSheet`(오늘 목표 상향 배치된 풀 컨텍스트 시트).

### 문제점
- **3초 판단의 종착점인 "운동 시작"이 홈에 없다**(발견 C). "오늘 가슴 확인 필요"를 읽고도 그 자리에서 운동을 못 켠다. 시선이 화면 하단 탭바로 분산.
- **블록1이 "확인 필요"라는 중립어라 행동이 안 떠오른다.** `ActionRow`(`index.tsx:185`)는 "확인 필요/기준" 태그만 줄 뿐, "이 종목을 오늘 운동에 넣어"라는 다음 동작 버튼이 없다. 탭하면 시트가 열려 또 읽어야 함(읽기→읽기).
- **헤더에 인사말만 있고("{요일} · 점진적으로 강해지는 중", `index.tsx:86`) 핵심 한 줄 요약이 늦게 나온다.** 진짜 3초 정보(오늘 추천 부위/다음 종목)는 스크롤 안쪽.
- **블록2 부위 볼륨 바가 정보 밀도 과다.** 출석 도트 + 부위별 트랙바 + 부족부위 텍스트가 한 카드에 겹쳐, "잘 가고 있나"를 한눈에 주기엔 시각 노이즈.
- **게이트 중 빈 화면**(`index.tsx:55` `return <View backgroundColor:'#000' />`)이라 온보딩 직후/로딩 시 새까만 깜빡임. 빈 상태 안내 부재.

### 개선안
- 블록1 각 행에 "오늘 운동에 추가" 또는 홈 상단 고정 "운동 시작" 버튼 추가(시나리오2 시작 시트로 연결).
- `ActionRow`를 "읽기"가 아니라 "행동" 카드로: 태그 대신 "→ 운동 시작" 버튼, 시트는 보조.
- 헤더 인사말을 "오늘 추천: 가슴·삼두 / 다음 막힌 종목 1개" 같은 실데이터 한 줄로 교체.
- 블록2를 "출석 N/4 + 부족 부위 1개"만 굵게, 나머지 바는 접힘/리포트로.
- 게이트/로딩 시 검은 화면 대신 스켈레톤 또는 "준비 중" 카드.

### 우선순위: High (시작 진입·행동 연결)

## 시나리오 4 — 정체/증량 이해

### 현재 흐름
1. 증량/정체는 stage로 표현(READY_TO_INCREASE / STALL_REVIEW). 홈 블록1·카드 배지(`index.tsx:218, 242`), 운동 중 초록 배너 + 다음 무게 자동 프리필(`workout.tsx:822-831`), 시트(`ExerciseGoalSheet.tsx`)에서 단계·비교·근거.
2. 시트 "오늘 목표" 박스가 헤더 바로 아래(`ExerciseGoalSheet.tsx:82`), 그 아래 현재단계 → 비교기준(직전/최고/PR + 신뢰도 칩) → 다음단계.

### 문제점
- **"확인 필요"가 증량과 정체를 한 묶음으로 중립화**(`index.tsx:68` needCheck = READY+STALL+DELOAD)했다. 의도는 좋으나(증량 영웅화 금지), 사용자 입장에선 "올려도 되는 건지 막힌 건지"가 한 라벨에 섞여 정작 가장 알고 싶은 이분(전진/정체)이 흐려진다. 카드 안으로 들어가야 색(초록/주황)으로 겨우 구분.
- **정체일 때 "그래서 뭘?"이 약하다.** 시트 다음단계 텍스트(`nextStep`)에 의존. 스펙은 "같은 조건 한 번 더 기록부터"인데, 정체 카드에 그 행동 버튼이 없어 텍스트를 읽고 스스로 판단해야.
- **비교 신뢰도 low일 때의 처리.** 시트는 low면 성공/실패 숨기지만(`workout.tsx:2286` `!lowComp`), 홈 카드는 todayTarget을 그대로 단정 노출. 기준 부족인데 "오늘 80kg" 같은 단정이 나가면 "비현실적 목표 금지" 철칙과 어긋날 여지.
- **증량 자동 프리필이 추가 시점에만 적용**(`confirmAddExercises:822`)된다. 이미 카드가 떠 있는 상태에서 증량 준비로 바뀌면 프리필이 안 되거나, 사용자가 무게를 손대면 그 뒤 자동값이 덮어쓰지 않는지(1탭 덮어쓰기 보장 여부)가 코드상 추가 경로에만 보여 일관성 의심.

### 개선안
- 블록1을 "↑ 증량 가능"과 "⚠ 정체 점검" 2그룹으로 시각 분리(중립 묶되 그룹 헤더로 구분). 색만이 아니라 라벨로.
- 정체 카드/시트에 "같은 무게로 오늘 운동에 추가" 원탭 액션 제공(스펙 출구 동작 직결).
- 홈 카드도 시트와 동일하게 comparability low면 todayTarget을 "기준 쌓는 중"으로 톤다운(단정 금지 일원화).
- 증량 프리필이 카드 표시 후에도, 신뢰도와 무관하게 항상 "1탭 덮어쓰기"되도록 배너에서 직접 트리거.

### 우선순위: Med (이해는 되나 "다음 행동" 연결과 단정 일관성이 약함)

## 시나리오 5 — 리포트 → 다음 행동

### 현재 흐름
1. 리포트 탭(`app/(tabs)/reports.tsx` → `components/report/ReportScreen.tsx`, showBack=false).
2. 구성: 다음 주 액션(히어로, 최대 3개 + "오늘 운동 시작하기" CTA) → 핵심 종목 변화 → 정체 점검 → 이번 주 출석 링 → 목표 부위 주간 세트 밴드.
3. 빈 상태: "아직 보여줄 게 없어요" + 운동하러 가기 CTA(`ReportScreen.tsx:113-122`).

### 문제점
- **이 화면이 5개 중 가장 "행동 연결"이 잘 돼 있다**(액션 히어로 + CTA가 `router.replace('/workout')`로 직결, `ReportScreen.tsx:142`). 장점.
- 다만 **CTA가 빈 운동을 안 만든다.** `onGoWorkout` → `/workout`로 가지만, 그 화면은 activeSession이 없으면 "내 루틴/지난 운동" 목록(시나리오2 시작 관문)으로 떨어진다. "오늘 운동 시작하기"를 눌렀는데 또 시작 방식을 골라야 하는 한 단계 추가.
- **"다음 주 액션"인데 진입은 항상 "이번 주" 탭.** 주가 끝나기 전엔 액션이 잠정적인데 사용자는 지금 행동으로 받아들일 수 있어 시점 혼동.
- **정체 점검 카드가 이름만 나열**(`ReportScreen.tsx:159-164` dot + name). 거기서 종목 상세/운동으로 가는 링크가 없어 "점검하라"는데 점검할 곳으로 못 간다.
- **출석/부위 밴드는 정보 제공형**이라 다음 행동과의 거리가 멀다. 리포트 하단부가 "읽고 끝"이 되기 쉽다.

### 개선안
- 액션 CTA가 곧장 빈 운동 세션을 생성(또는 추천 종목 프리필된 세션)해 한 탭으로 입력 화면 진입.
- 정체 점검 항목 각각을 탭 가능하게 → 종목 상세 또는 "같은 무게로 운동 추가"로.
- "다음 주 액션" 라벨을 "이번 주 마무리/다음 액션" 등 시점 명확화.
- 출석/밴드는 접힘 또는 "부족 부위 1개 채우기" 같은 행동 한 줄로 마감.

### 우선순위: Med (구조는 좋음, CTA 한 단계 단축 + 정체→행동 링크만)

## 완료 후 가장 큰 UX 마찰 8줄

1. 홈에 "운동 시작" 진입점이 0개라(오직 탭바 play) "오늘 뭐 할지" 읽고도 그 자리에서 운동을 못 켠다 — 3초 판단의 종착이 끊긴다(`index.tsx`, 발견 C).
2. 온보딩 점진과부하 목표(목적·주간횟수·증량폭)를 이후 수정할 화면이 없다 — 설정 "목표 설정"은 체중/체지방/휴식이라 코어 값이 영구 고정된다(발견 A).
3. 설정 "온보딩 초기화"가 실제론 AI 프로필만 지우고 홈을 막는 onboarding 게이트는 안 건드린다 — 라벨-동작 불일치 버그(발견 B).
4. 온보딩 4단계 "증량 판정 시점" 3지선다가 스펙(two_sessions 고정·묻지 말 것)에 역행하고 default가 `single`이라, 첫 사용자에게 불필요한 전문 결정을 강요한다(`onboarding.tsx:54`).
5. 종목 추가가 부위→장비→브랜드→목록 다단 드릴다운이라, 같은 종목을 반복하는 헤비유저가 매 운동 4탭 깊이를 다시 판다(최근·즐겨찾기가 기본 진입에서 가려짐).
6. 노력도 4버튼이 마지막 세트 후 0.8초 지연 자동 팝업이라, 바로 다음 세트로 넘어가려는 흐름을 모달이 가로채고 드롭세트 등에서 타이밍이 어긋난다(`workout.tsx:937`).
7. 홈 블록1이 증량(전진)과 정체를 "확인 필요" 한 라벨로 묶어, 가장 알고 싶은 전진/정체 이분이 카드 안 색으로만 구분돼 한눈에 안 잡힌다(`index.tsx:68`).
8. 종목 상세가 스펙상 "축소판"인데 5지표 토글+히트맵+LLM코치+AI리포트+체크리스트가 그대로 남아, MVP 제외 잔재가 미완성·과부하 인상을 준다(`exercise/[name].tsx`).
