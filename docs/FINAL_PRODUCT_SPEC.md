## 0. 사용자 확정 오버라이드 (2026-06-29) — 최우선

Critic이 MVP에서 제외 권고한 3건을 사용자가 검토 후 **모두 포함으로 확정**했다. 본문에서 충돌하는 서술은 이 절이 우선한다.

1. **노력도(RIR) 버튼 = MVP 포함, 끝까지 연결.** 마지막 작업 세트에 4버튼(여유 많음 / 2~3개 남음 / 거의 한계 / 실패)을 받고, WorkoutSet에 노력도 컬럼을 추가해 저장하고, 엔진의 증량 준비 판정(rpe 트리거)과 성공 조건에 실제로 반영한다. "받고 버리는 죽은 입력"이 아니라 "받아서 쓰는 입력"으로 만든다. 단 숫자 강요 금지·스킵 가능·핵심 종목 마지막 세트에만.
2. **리포트 = MVP 포함, 결정론으로 개선.** LLM 경로는 빼되 리포트 화면 자체는 유지하고, 엔진/통계 값(이번 주 출석·목표 부위 세트·핵심 종목 변화·정체 점검·다음 주 액션 1~3개)으로 채운다. 정체 정의는 엔진 stalling 단일 소스로 통일하고 리포트도 그 값을 쓴다(plateauWeeks 이중 정의 폐기).
3. **Chat 탭 = MVP 유지.** AI 채팅은 보조 질의응답으로 남긴다. 단 점진적 과부하 핵심 처방(오늘 목표·증량·정체)은 결정론 엔진이 단일 소스이고, AI는 그 위에서 "왜"를 설명하는 보조역만. (종목 상세의 구조화 AI 코치 리포트·체크리스트는 별개로 MVP 제외 유지 — Chat 탭만 유지)

## 1. 앱 한문장 컨셉

이미 가진 무게·횟수 기록만으로 "오늘 이 종목에서 뭘 하면 되는지"를 3초 안에 알려주는, Strong을 더 편하게 대체하는 점진적 과부하 코칭 앱.

## 2. 타겟 사용자

중급+ Strong 헤비유저 본인 1명. 자기 루틴이 고정돼 있고, 매일 직접 운동 기록을 남기며, 무게를 언제 올릴지·정체인지 판단하는 데 시간을 쓰고 싶지 않은 사람. 초급/다인 대응·자가신고 경력·코칭 대화는 타겟이 아니다.

## 3. 핵심 사용자 문제

1. Strong은 기록은 잘 되지만 "오늘 이 종목에서 무게를 올릴지, 반복을 더 할지"를 매번 사용자가 직접 판단해야 한다.
2. 헬스장에서 지난 기록을 찾아 비교하느라 시간이 든다(3초 안에 안 보임).
3. 정체인지 그냥 컨디션 나쁜 날인지 구분이 어렵고, 막혔을 때 무엇을 할지 막막하다.
4. 증량 준비가 됐는데도 무게를 손으로 다시 계산해 입력해야 한다.

## 4. 해결 방식

결정론적 점진적 과부하 엔진(백엔드 `ProgressionEngine`)이 직전 비교 기록을 기준으로 종목별 단계와 오늘 목표를 매번 재계산한다. 사용자는 무게·횟수·완료만 입력하고, 직전값 프리필·증량 준비 시 다음 무게 자동 채움으로 입력을 더 줄인다. 정체는 한 곳(엔진)에서만 선언하고, 막혀도 즉시 감량하지 않고 "같은 조건 한 번 더 기록"부터 권한다. LLM·차트 과다·노력도 입력은 MVP에서 배제한다.

## 5. 목표 시스템

- 온보딩 1회: 목적(hypertrophy/strength/fatloss/endurance) · 주간 운동 횟수 · 상/하체 증량폭(kg). 이 3개로 rep range·증량폭·주간 출석/부위 목표가 전부 도출된다.
- 종목별: 역할(core/support/log_only)과 분류(barbell/machine/isolation/bodyweight)는 장비·부위에서 자동 배정, 사용자는 틀렸을 때만 수정.
- 증량 트리거는 **two_sessions 고정**(2세션 연속 상단 달성 시 증량 준비). 온보딩에서 묻지 않는다.

## 6. 사용자에게 받는 값

- 온보딩(평생 1회, 3개): 목적 · 주간 횟수 · 상/하체 증량폭.
- 종목 추가 시: 종목 선택 1번(역할은 자동, 수정만).
- 세트마다: 무게 · 횟수 · 완료(시간기반은 초). 세트 타입은 선택.
- **핵심 종목 마지막 작업 세트: 노력도 버튼 1개**(여유 많음 / 2~3개 남음 / 거의 한계 / 실패). 숫자 강요 금지·스킵 가능. 저장 후 엔진 증량 판정에 반영(§0-1).
- 운동 시작/종료: 시작 방식 탭 1번(최근 세션 불러오기 / 빈 운동), 종료는 추가 입력 0.
- **받지 않는 것**: 종료 후 컨디션 · 증량 트리거 선택 · 부위/경력 직접 입력. (노력도는 §0-1로 포함 확정)

## 7. 앱이 자동 계산하는 값

직전/최고/전체 PR 기록 · 오늘 목표(todayTarget) · 성공 조건 · 남은 총 반복수(targetTotalReps) · 다음 목표 무게(nextTarget, 머신/고립은 증량폭 절반·최소 0.5kg) · 현재 단계(stage) · 증량 준비 여부 · 비교 신뢰도 · 정체 여부(엔진 stalling 단일 정의) · 세션 평가(개선/유지/재도전/기준완성/비교보류) · 부위별 주간 세트·부족분 · 출석 진행 · 세트별 1RM(Epley) · PR 감지 · 워밍업 자동 생성 · 휴식 시간. 사용자는 이 값들을 손으로 입력하지 않는다.

## 8. 점진과부하 기본 방식

**Double Progression(이중 진행)**이 단일 기본 엔진이다. 무게 고정 상태에서 직전 총 반복수 + 작은 bump(+1~2)로 반복을 쌓고, 반복 범위 상단을 세트 수만큼 채우면(two_sessions 트리거 충족) 소폭 증량한다. 오늘 목표는 항상 직전 비교 기록 기준 + 작은 bump로만 계산하며, 장기 목표(예 15회)를 오늘 목표로 절대 산출하지 않는다. 최고 기록은 현재 위치 판단용, 전체 PR(1RM)은 동기부여용으로 역할을 분리한다.

## 9. 운동유형별 진행 방식

4개 분류는 별개 방식이 아니라 같은 Double Progression을 (반복 범위 · 증량폭 · 정체 출구) 3파라미터로 분기한 것.

| 분류 | 반복 범위 | 진행 | 증량폭 | 정체 출구 |
|---|---|---|---|---|
| BARBELL_COMPOUND | 비대 6~12 / 근력 3~6 | 총 반복 +1~2 → 상단 달성 시 증량 | 상 +2.5 / 하 +5(사용자 값) | 세트 추가 → 재차 막히면 재기록·검토 |
| MACHINE_OR_CABLE | 8~12 | 총 반복 +1~2 → 상단 달성 시 증량 | 마이크로로딩(절반, 최소 0.5kg) | 세트 추가 우선 |
| ISOLATION | 8~15 | 자세·반복 우선, 보수적 증량 | 마이크로로딩(절반, 최소 0.5kg) | 세트 추가 우선 |
| BODYWEIGHT | 5~12 | 총 반복수 +1 | 임계 후 가중(MVP 제외) | 같은 목표 한 번 더 |

> Critic 반영: 머신 bump를 [2,3]에서 **+1~2로 통일**(작은 bump 철칙 일관), 맨몸 단기목표 ×1.5 산식은 단순화.

## 10. 핵심·보조·기록만 구분

- **core(핵심)**: 단계 머신 적극 추적. 오늘 목표·증량 신호 모두 제공. 홈/운동 중 우선 노출.
- **support(보조)**: 카드는 보여주되 부위 볼륨·주간 균형 우선. 무리한 증량 처방 자제(약한 스타일).
- **log_only(기록만)**: 단계 분석·세션 평가 제외, 단순 기록. 무게0·증량 비대상 종목은 기본 role을 support/log_only로 보수적 자동 배정(Critic 6-2 반영).

## 11. 홈 역할

켜자마자 "오늘 뭐부터 + 이번 주 잘 가나"만. **9섹션 → 3블록으로 압축(Critic 8-1)**:
1. 오늘 행동이 필요한 종목(증량 준비 + 정체 점검을 "확인 필요"로 중립 묶음 — 증량 단독 영웅화 금지, Critic 6-1).
2. 이번 주 1줄(출석 + 부족 부위 — 부위 볼륨을 증량 종목과 동급 이상으로, Critic 7-2).
3. 종목 리스트(핵심 펼침 / 보조·기록만 기본 접힘).
주간 코멘트·개선 도트바·오늘운동하면 카드·추천 루틴은 제거 또는 "이번 주" 카드로 병합.

## 12. 종목 바텀시트 역할

카드를 탭하면 열리는 풀 컨텍스트. **오늘 목표 박스를 헤더 바로 아래로 상향**(현재 3번째 → 2번째). 순서: 헤더 → 오늘 목표(가장 큼: todayTarget + successCondition + caution) → 현재 단계 → 비교 기준(직전/최고/전체 PR + 비교 신뢰도 칩 + 근거) → 다음 단계 → 단기·장기 목표(있을 때만) → 역할 수정. 비교/단계는 "근거"로 아래에 둔다.

## 13. 운동 중 화면 역할

세트 입력 화면. 손은 바쁘고 눈은 0.5초. **goal banner를 1줄로 축소(Critic 9-2)**: "오늘 80kg 총 24회 / 남은 8회". 단계 라벨·비교 보류·성공조건은 카드 탭(시트)으로 미룬다. 증량 준비면 배너를 초록으로 + **다음 무게 자동 프리필(언제든 1탭 덮어쓰기 보장, Critic 9-3)**. 비교 신뢰도 low면 성공/실패 단정 금지. log_only는 배너 숨김. **핵심 종목의 마지막 작업 세트 완료 시 노력도 4버튼(§0-1)** — 한 번 탭하면 끝, 스킵 가능.

## 14. 운동 종료 화면 역할

추가 입력 0. 읽기 전용 요약: 세션 한 줄(볼륨/세트/종목 수, PR면 신기록) + 종목별 평가 칩(기준완성/개선/유지/재도전/비교보류) + 다음 목표 1줄. 비교 보류는 "하락"으로 표기하지 않는다. 컨디션 입력 없음.

## 15. 리포트 역할

**MVP 포함 — 결정론으로 개선(§0-2).** 리포트 탭은 유지하되 LLM 브리핑을 빼고 엔진/통계 값으로 채운다: 이번 주 출석 · 목표 부위 주간 세트 · 핵심 종목 변화(개선/유지/정체) · 정체 점검 종목 · 다음 주 액션 1~3개. 숫자 나열이 아니라 "다음 행동"으로 연결. **정체는 엔진 stalling 단일 정의만 사용**(리포트의 plateauWeeks 이중 정의 폐기 — 홈·시트·리포트가 한 목소리). 주간 LLM 브리핑은 후속.

## 16. AI 역할

**점진적 과부하 핵심 처방은 100% 결정론 엔진(단일 소스).** 오늘 목표·증량·정체·성공조건·다음단계는 엔진 카피(todayTarget/successCondition/nextStep/comparisonReason)로만 낸다. **Chat 탭은 유지(§0-3)** — 보조 질의응답 및 "왜"의 설명역. 단 Chat이 처방 숫자를 새로 만들지 않고 엔진 값을 인용한다. 종목 상세의 구조화 AI 코치 리포트·체크리스트·buildNextWeekGoals는 MVP 제외(혼란·정체정의 충돌 방지).

## 17. MVP 포함 기능

1. 세트 입력(무게·횟수·완료) + 직전값 프리필.
2. 5 stage 결정론 엔진(NEED_BASELINE/BASELINE_CHECK/BUILD_REPS/READY_TO_INCREASE/STALL_REVIEW) + 직전 비교 기록 기준 오늘 목표(작은 bump +1~2 통일).
3. 운동 중 1줄 목표 배너 + 증량 준비 시 다음 무게 자동 프리필(1탭 덮어쓰기).
4. **노력도(RIR) 버튼 풀 연결(§0-1)**: 핵심 종목 마지막 세트 4버튼 → WorkoutSet 저장 → 엔진 rpe 트리거·성공조건 반영.
5. 종목 바텀시트(오늘 목표 상향).
6. 압축 홈 3블록(부위 볼륨 동급 이상, 행동 필요 종목 중립 묶음).
7. 운동 종료 읽기전용 요약(평가 칩 + 다음 목표 1줄).
8. **리포트(결정론, §0-2)**: 출석·부위 세트·핵심종목 변화·정체 점검·다음 주 액션 1~3개. LLM 없음.
9. **Chat 탭 유지(§0-3)** — 보조 질의응답(처방은 엔진 인용).
10. 온보딩 3개(목적·주간횟수·상하체 증량폭, 트리거 two_sessions 고정).
11. 기준 부족=분석 금지·비교 보류는 하락 표기 금지(집계 모수에서 baseline/deferred 제외).
12. 정체 단일 정의(엔진 stalling)·첫 정체는 재기록 우선·즉시 감량 금지.
13. 종목 상세 축소판(1RM 추이 1차트 + 최근 기록 표 + 바텀시트 요약).

## 18. MVP 제외 기능

1. 종료 후 컨디션 입력 · 온보딩 증량 트리거 선택 UI · 추천 루틴 자동생성 · 주간 코멘트.
2. 주간 LLM 브리핑(리포트 화면은 포함하되 LLM 텍스트 생성만 제외).
3. AI 코치 대화 "대화로 풀기" 처방 생성 · buildNextWeekGoals(Chat 탭 자체는 유지, 처방 생성만 제외).
