# GymTracker · UI/UX 개선 10단계 로드맵

> 살아있는 추적 문서. 화면 기준은 [FLOWS.md](FLOWS.md), 캡처는 [screenshots/](screenshots/README.md).
> 앱 강점 유지: "애널리스트" 컨셉 · CARBON 다크(블랙+레드) · 종목 카드 그리드.

## 진행 방법 (각 단계 공통)
1. 탐색(현 화면 감사) → 계획 요약 → **STOP·승인** → 구현 → **시뮬레이터 캡처 검증** → 커밋.
2. 순서: 기반 → 핵심 플로우 → 폴리시 → 품질 (뒤 단계가 앞 토큰·컴포넌트 재사용).
3. 측정: 단계마다 전/후 캡처 + 핵심 지표 1개.

## 색/모션 원칙
레드 = 액션·네비 / 초록 = 양호(↑·PR) / 주황 = 주의. 파랑 미사용.

## 10단계 (체크리스트)

### 기반
- [x] **1. 디자인 시스템 감사 + 컴포넌트 인벤토리** — `components/ui/BottomSheet` 신설(SortSheet·그룹시트 치환), `RADIUS.card` 토큰, `docs/DESIGN.md`(토큰 단일소스 + hex→토큰 규칙). 감사: TYPE/SPACE/RADIUS 사용 0회·fontSize 615/여백 981 하드코딩 확인. ⚠ 대량 hex 스윕은 blind 치환이 JSX 깨뜨려 보류 → 화면별 점진(2~10단계). ref: Linear, Things.
- [ ] **2. 정보 위계·타이포·글랜서빌리티** — 화면별 1차/2차/3차 재배치, "한눈에 핵심 1개". ref: Whoop, Oura.

### 핵심 플로우
- [ ] **3. 운동 로깅 속도(최우선)** — 세트 입력/완료/휴식 마찰 제거, 자동 포커스·이전값·한 손. ref: Strong, Hevy. 지표: 1세트 기록 탭 수·시간 30%↓.
- [ ] **4. 브리핑 홈 — 히어로 + 점진 공개** — 헤드라인+처방 영웅화, 나머지 단계적. ref: Whoop, Toss.
- [ ] **5. 리포트 — 데이터 스토리텔링·밀도** — 주의→추세→구성→몸 내러티브, 차트 일관, 빨강 과부하 점검. ref: Oura, Apple Health.

### 폴리시
- [ ] **6. 로딩·전환·스켈레톤·모션** — SWR 캐시 우선 전 화면, 공용 `<Skeleton/>`, 전환 모션 통일. ref: Toss, Arc.
- [ ] **7. 빈/오류/온보딩 상태** — 일러스트+행동유도, 첫 사용자 가이드. ref: Duolingo, Headspace.
- [ ] **8. 마이크로 인터랙션·햅틱·축하** — PR 축하, 완료 피드백, 제스처 다듬기. ref: Apple Fitness, Strong.

### 품질·반복
- [ ] **9. 접근성·대응성·한 손 도달** — 대비·동적 폰트·탭 타깃 44pt·VoiceOver·엄지 영역. ref: Apple HIG.
- [ ] **10. 측정·사용성 테스트·디자인 부채 루프** — 핵심 지표·테스트 주기·부채 추적. ref: Superhuman, Linear.
