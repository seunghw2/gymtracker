# GymTracker 화면 명세 (UX_SCREEN_SPEC)

> 작성: UX Product Agent. 근거 = APP_AUDIT.md + 실제 코드(`index.tsx`, `ExerciseGoalSheet.tsx`, `workout.tsx`, `db/api/overload.ts`). 코드 미수정.
> 타깃 = 본인 1명(Strong 헤비유저, 중급+). 목표 = Strong을 더 편하게 대체하는 점진적 과부하 코칭.

## 대원칙 (모든 화면 공통)

1. **헬스장 3초 룰**: 화면을 켜면 3초 안에 "오늘 이 종목에서 뭘 하면 되는지"가 보여야 한다. 분석은 그다음.
2. **홈에 분석 과다 금지**: 홈은 "오늘 할 일 + 진척 한눈"까지만. 차트·원인분석·히스토리는 종목 상세/리포트로 미룬다.
3. **운동 중엔 설명보다 숫자**: "오늘 목표 50kg 총 39회 / 현재 30회 / 남은 9회". 문장은 짧게, 숫자는 크게.
4. **비교 조건 다르면 단정 금지**: 세트 수가 2+ 차이거나 기록 부족이면 "개선/하락"을 말하지 않고 "비교 보류"로 표기.
5. **기준 부족이면 분석 대신 기준 생성**: 기록 0~1개면 "기준 만들기" 단계로 안내. 비현실적 목표(3회 만에 +15회 등) 금지.
6. **우선순위 + 접기/펼치기로 과부하 방지**: 지금 행동에 필요한 카드만 펼치고, 참고용은 접거나 하단에.

## 0. DTO 가용 필드 (화면이 쓸 수 있는 데이터)

`ExerciseGoalDto`(`db/api/overload.ts`) 실제 필드 — 화면 설계는 이 안에서만 한다(임의 신규 데이터 금지):
- 식별/역할: `exerciseName`, `muscleGroup`, `ruleType`(barbell_main/machine_cable/bodyweight/isolation), `role`(core/support/log_only)
- 단계: `stage`(NEED_BASELINE/BASELINE_CHECK/STALL_REVIEW/BUILD_REPS/READY_TO_INCREASE 등), `stageLabel`
- 오늘: `todayTarget`(문구), `targetTotalReps`(운동 중 남은 반복수 계산용), `successCondition`, `caution`, `nextStep`, `nextCondition`
- 비교: `lastRecord`/`lastRecordDate`, `bestRecord`, `allTimePr`, `comparability`(high/medium/low), `comparisonReason`
- 보조목표: `shortTermTarget`, `longTermTarget`

`WeeklySummaryDto`: `attendance{done,target}`, `improvements{hasData,done,total}`, `bodyPartVolumes[{part,korPart,currentSets,targetSets}]`, `bodyPartGaps`, `todayPlan`(문구), `comment`(규칙 기반 코멘트).

> 운동 종료 평가(IMPROVED/MAINTAINED/MISSED/BASELINE_CREATED/COMPARISON_DEFERRED)는 `getSessionEval(sessionId)` 응답에서 온다(workout.tsx 종료 플로우).

## 1. 홈 (`app/(tabs)/index.tsx`)

### 목적
켜자마자 "오늘 뭐부터" + "이번 주 잘 가고 있나"를 본다. 종목 카드는 **행동이 필요한 순서**로.

### 표시 순서 (위→아래, 우선순위)
1. 인사 + 한 줄 코멘트 (`summary.comment`) — 1줄, 없으면 생략
2. **오늘 추천 / 운동 시작** (CTA) — 화면 최상단 고정 행동
3. **증량 준비 종목** (READY_TO_INCREASE) — 가장 먼저, 초록 강조. "오늘 올릴 종목"
4. **기준 기록 필요 종목** (NEED_BASELINE/BASELINE_CHECK) — "오늘 기준부터 만들 종목"
5. **이번 주 출석** 도트바 (`attendance`)
6. **목표 부위 주간 세트** 진행바 (`bodyPartVolumes`)
7. **오늘 운동하면 채워지는 목표** (`todayPlan` + 출석 +1)
8. 핵심 종목 그룹 (전체) → 보조 → 기록만 (접기 가능)
9. 이번 주 개선 도트바 (`improvements`) — 데이터 없으면 "다음 운동부터 추적"

> 변경 핵심: 현재(audit)는 [이번주목표 → 부위세트 → 오늘운동하면 → 핵심/보조/기록만] 순서로, 증량 준비/기준 필요 종목이 핵심 그룹 안에 묻혀 있다. **"증량 준비"와 "기준 필요"를 그룹에서 끌어올려 최상단 액션 섹션으로 승격**한다. 헬스장에서 첫 화면에 "오늘 올릴 것 / 오늘 기준 잡을 것"이 바로 보여야 함.

### 접기/펼치기
- 증량 준비 / 기준 필요 = 항상 펼침(0건이면 섹션 자체 숨김).
- 핵심 종목 = 펼침. 보조/기록만 = 기본 접힘(헤더 탭으로 펼침).
- 출석·부위세트·오늘운동하면 = 한 카드로 묶어 "이번 주" 카드 1장으로 압축 가능(스크롤 절감).

### 종목 카드 (GoalCard) 사양
한 카드에서 보여줄 것 / 순서:
1. **종목명** (가장 큼)
2. **역할 + 단계 배지** 한 줄 (핵심/보조/기록만, 단계는 색배지: 증량준비=초록 / 기준필요=회색 / 정체=주황 / 진행중=강조색)
3. **지난 기록** (1줄, `lastRecord`. 기준필요면 생략)
4. **오늘 목표** (`오늘 {todayTarget}`, 가장 강조되는 텍스트)
5. **다음 단계 조건 요약** (`caution ?? nextCondition`, 1줄, compact 카드는 생략)

> 재활용: 현재 `GoalCard`(index.tsx:190-224)가 이미 이 구조에 매우 가깝다(이름·룰배지·단계배지·지난기록·오늘목표·조건). 바꿀 것 = (a) 카드가 그룹이 아니라 "액션 섹션(증량준비/기준필요)"으로도 재배치될 수 있게 묶음 기준 변경, (b) 룰타입 라벨(`pcardType`)은 보조 정보이므로 시각 우선순위를 낮춤.

## 2. 종목 카드 (재사용 컴포넌트 명세)

홈/액션섹션/(필요시)종목 허브에서 같은 카드를 쓴다. 7개 질문 중 카드에서 답하는 것:
- **무엇을**: 종목명 + 역할
- **지금 단계**: 단계 배지
- **오늘 뭘 하면**: 오늘 목표 1줄
- **기준 필요한가**: NEED_BASELINE이면 "기준 필요" 배지 + "오늘 기준 만들기" 문구로 대체
- **다음 단계 조건**: 1줄 요약

나머지 4개 질문(어떤 기록 기준 / 최고 비교기록 / 전체 PR / 비교 신뢰)은 **카드에서 안 보이고 바텀시트로 미룬다**(과부하 방지).

## 3. 종목 바텀시트 (`components/ExerciseGoalSheet.tsx`)

### 목적
카드를 탭하면 열리는 "이 종목 풀 컨텍스트". 7개 질문 전부에 답한다. 단, **오늘 목표를 가장 크게**, 비교 근거는 그 아래.

### 섹션 순서 (7개 질문 매핑)
1. 헤더: 종목명 + 역할 배지 + 부위·룰타입 메타
2. **오늘 목표** (가장 큰 박스) — `todayTarget` + `successCondition`(=오늘 뭘 하면 성공) + `caution`
   - → 질문 "오늘 뭘 하면 성공"
3. **현재 단계** — `stageLabel`
   - → 질문 "지금 어떤 단계"
4. **비교 기준** — 직전(`lastRecord`, 탭하면 그날 세션 미리보기) / 최고(`bestRecord`) / 전체 PR(`allTimePr`) + 비교 신뢰도 칩(`comparability`) + 근거(`comparisonReason`)
   - → 질문 "어떤 기록 기준 / 최고 비교기록 / 전체 PR / 이 비교 믿을만한가"
5. **다음 단계** — `nextStep ?? nextCondition` (성공 시/여유 시/실패 시 분기 문구)
   - → 질문 "성공하면 다음 / 실패하면"
6. **단기·장기 목표** — `shortTermTarget`/`longTermTarget` (있을 때만, 맨몸 등)
7. **역할** — 역할 설명 + core/support/log_only 변경
   - → 역할 설명

> 변경 핵심: 현재(ExerciseGoalSheet.tsx)는 [현재 단계 → 비교 기준 → 오늘 목표 → ...] 순서로 **오늘 목표가 세 번째**다. 3초 룰에 맞춰 **오늘 목표 박스를 헤더 바로 아래(2번)로 끌어올린다**. 비교 기준/단계는 "근거"이므로 그 아래로. 나머지 구조·컴포넌트(`Section`, todayBox, 비교행, 신뢰도 칩, 역할 편집)는 그대로 재활용.

## 4. 운동 중 (`app/workout.tsx` 종목 카드 내 goal banner)

### 목적
세트 입력 화면에서 손은 바쁘고 눈은 0.5초. "지금 이 종목 목표 / 얼마 했나 / 몇 회 남았나 / 마지막 세트 성공조건"만.

### 표시 (이미 banner로 일부 구현됨 — workout.tsx:1706-1739)
종목 카드 헤더 아래 한 줄짜리 배너:
1. 단계 라벨 (`stageLabel`) + 비교 보류 시 "· 비교 보류"(`comparability==='low'`)
2. **오늘 목표** (`todayTarget`) — 큰 글씨
3. 진행: `caution` 있으면 caution, 없으면 `targetTotalReps` 기준
   - 미달: "현재 {doneTopReps}회 · 목표까지 {remaining}회"
   - 달성: "✓ 오늘 목표 달성 ({doneTopReps}회)"
4. log_only 역할은 배너 자체 숨김. 보조 역할은 약한 스타일.

### 변경/보강 포인트
- **마지막 세트 성공조건 노출 강화**: 현재 진행 텍스트는 "목표까지 N회"만 보여준다. 마지막 작업 세트를 입력하는 순간 "이번 세트 9회면 성공"(=`successCondition` 또는 remaining 기반)을 **세트 행 옆/배너에 명시**. audit 격차(13-2: 운동 중 마지막세트 성공조건이 약함) 보완.
- **증량 준비 신호**: stage가 READY_TO_INCREASE면 배너를 초록으로, 다음 무게(`todayTarget`의 무게)를 강조. (자동 프리필은 엔진/입력 영역 작업이라 본 명세 범위 밖 — 표시까지만.)
- **비교 불가 정직 표기**: `comparability==='low'`면 "비교 보류"를 유지하고 성공/실패 단정 문구를 띄우지 않는다.

## 5. 운동 종료 (`workout.tsx` → proceedFinish 요약 시트)

### 목적
끝나고 "오늘 뭐가 좋아졌나 + 다음에 뭐 할까"를 종목별 1줄로. 숫자 나열 대신 평가 라벨.

### 표시
1. 세션 요약 한 줄: 볼륨/세트/종목 수, PR 있으면 "🏆 신기록".
2. **종목별 평가 칩** (`getSessionEval` 결과):
   - 기준 생성됨(BASELINE_CREATED) → "기준 완성 ✓"
   - 개선(IMPROVED) → "개선 ▲"
   - 유지(MAINTAINED) → "유지"
   - 실패(MISSED) → "다음에 재도전"
   - 비교 보류(COMPARISON_DEFERRED) → "비교 보류"(개선/하락 단정 안 함)
3. **다음 운동 목표** 종목별 1줄 (`nextStep`/다음 todayTarget) — 증량 정착/유지/감량 등.
4. 닫기 = 홈으로.

### 와이어프레임
```
───── 운동 완료 ─────
52분 · 18세트 · 5종목 · 🏆 신기록 1
─────────────────────────────
Lat Pulldown     개선 ▲   다음: 52.5kg 정착
Bench Press      유지     다음: 80kg 한 번 더
Incline DB       기준 완성✓ 다음: 비교 시작
Leg Press        비교 보류  세트 수 달라 판단 보류
─────────────────────────────
[ 확인 ]
```

## 6. 리포트 (`app/(tabs)/reports.tsx` → `components/report/ReportScreen`)

### 목적
"숫자 대시보드"가 아니라 "이번 주 점검 + 다음 주 액션 1~3개". 모든 항목은 **다음 행동으로 연결**.

### 표시 순서
1. **이번 주 한 줄 요약** — 출석·핵심종목 개선 수 (예: "출석 4/5, 핵심 3종목 중 2개 개선")
2. **목표 부위 세트** — 부위별 충족/부족, 부족 부위 1개 강조 → "다음 주 등 +4세트"
3. **핵심 종목 개선 현황** — 종목별 개선/유지/정체 칩 (숫자 차트 아님)
4. **정체 점검 종목** — STALL/plateau 종목 1~2개 + "원인은 종목 상세에서" 링크
5. **다음 주 액션 1~3개** (가장 강조) — 예: ① 등 볼륨 +4세트 ② Lat Pulldown 52.5kg 도전 ③ Bench 정체 — 디로드 고려
6. 정직 모드: 데이터 0 → "기록을 더 쌓아야 분석 가능", 3주 미만 → 신뢰도 낮음 배지.

> 변경 핵심: 리포트는 stats 기반 AI 브리핑(audit 6)이고 overload 단계머신과 별개 시스템이다. UX 관점에선 **둘이 충돌하지 않게 "정체"를 한 목소리로** 내야 한다(audit 14-4). 표시는 "분석 텍스트"보다 **맨 아래 액션 1~3개**를 시각적으로 가장 크게. 차트/수치 나열은 접거나 종목 상세로 위임.

## 핵심 6줄
1. 홈 첫 화면을 "오늘 올릴 종목(증량 준비) / 오늘 기준 잡을 종목(기준 필요)"부터 보여주도록, 이 둘을 핵심 그룹에서 끌어올려 최상단 액션 섹션으로 승격한다.
2. 종목 바텀시트는 오늘 목표 박스를 헤더 바로 아래로 올려, 단계·비교 근거보다 "오늘 뭘 하면 되나"를 먼저 보게 한다(현재는 세 번째).
3. 운동 중 화면은 이미 있는 goal banner를 살리되, "마지막 세트 N회면 성공" 성공조건을 세트 행에 인라인으로 노출해 헬스장 3초 룰을 채운다.
4. 비교 신뢰도 low는 모든 화면에서 "비교 보류"로 통일하고 개선/하락을 단정하지 않으며, 기록 0~1개는 분석 대신 "기준 만들기"로 안내한다.
5. 운동 종료는 종목별 평가 칩(기준완성/개선/유지/재도전/비교보류) + 다음 목표 1줄로 정형화한다.
6. 리포트는 숫자 나열을 줄이고 "다음 주 액션 1~3개"를 화면에서 가장 크게 두며, 정체 정의를 단계머신과 한 목소리로 맞춘다.
