# GymTracker 화면별 인터랙션 명세 (Screen Spec)

> 최종 업데이트: 2026-06-18
> **목적**: 각 화면이 제공하는 기능 + 모든 버튼/제스처의 동작을 기술한다. **UI/UX 개편을 의뢰할 때의 "현재 동작" 기준선** — 무엇을 보존해야 하고 무엇을 바꿀 수 있는지의 출발점.
> 사용법은 맨 끝 [개편 의뢰용 가이드](#개편-의뢰용-가이드) 참고. 기능 충족 여부 점검은 [REQUIREMENTS.md](REQUIREMENTS.md).
> 표기: `요소 → 동작`. 모든 화면은 다크(CARBON) 테마, 한국어.

## 전역 요소

- **하단 탭바**(`CustomTabBar`): `브리핑 · 기록 · 종목 · [리포트] · Chat` 5칸. 리포트는 합성 탭(누르면 `/ai/reports`로 push, (tabs) 밖 스택). Chat 탭엔 안읽음 배지. 설정 탭은 바에 없음(브리핑 톱니로 진입). 활성=레드 인디케이터.
- **운동 중 배너**(`ActiveWorkoutBanner`): 진행 중 세션이 있고 운동 탭이 아닐 때 탭바 위에 도킹. 탭 → 운동 화면 복귀.
- **휴식 타이머 엔진**(`RestTimerEngine`): UI 없음. 화면 어디서든 휴식 카운트다운·사운드·로컬알림·자동 종료 처리.

## 1. 로그인 `app/(auth)/login.tsx`
- **목적**: 소셜 로그인 진입(이메일 회원가입 화면은 제거됨 — 소셜 only).
- **진입**: 미인증 상태 부트스트랩.
- **요소·동작**:
  - `카카오로 시작하기` → 카카오 OAuth → 토큰 발급 → 앱 진입.
  - `Apple로 계속하기`(기기 지원 시만 노출) → Apple 인증.
  - `Google로 시작하기`(클라이언트 ID 설정 시만 노출) → Google idToken 인증.
  - 실패 시 Alert로 사유 표시.
- **상태**: 각 버튼 로딩 스피너.

## 2. 브리핑(홈) `app/(tabs)/index.tsx`
> 상세 인터랙션 문서: [screens/briefing.md](screens/briefing.md) (데이터 흐름·상태 머신·모달·엣지케이스)
- **목적**: AI 주간 브리핑 요약 + 빠른 운동 시작 + 체중 입력.
- **구성**(위→아래): 헤더(브리핑 + ⚙️) · 날짜칩 · 큰 헤드라인 · 처방 카드 · 부위별 볼륨 카드 · 핵심 지표 3 · 운동 시작 버튼.
- **요소·동작**:
  - ⚙️ 톱니 → 설정 탭.
  - **헤드라인** 탭 → `/ai/reports`.
  - **처방 카드** 탭 → `/ai/reports`. (프로필 미설정이면 카드가 "시작하기"로 바뀌고 탭 → `/ai/intake`)
  - **부위별 볼륨 카드**(하드세트·빈도/주) 탭 → `/ai/reports`. 부족 부위는 빨강.
  - 지표 3: `이번 주` 운동수 / `스트릭` 연속일 / `목표까지 kg`(또는 "체중 입력") — 세 번째 탭 → **체중 입력 모달**.
  - **운동 시작** → `/workout`(모달).
  - 당겨서 새로고침.
  - 리포트 생성 중이면 헤드라인 영역에 오비탈 로딩(`BriefingLoading`, 실시간 %), 2초 폴링으로 완료 감지.
- **체중 입력 모달**: 룰러로 체중(kg) 선택, 체지방%·허리cm 선택 입력 → `저장`. 오버레이/저장 시 오늘은 다시 안 띄움.

## 3. 기록 `app/(tabs)/calendar.tsx`
- **목적**: 운동 히스토리 — 타임라인 / 월별 캘린더.
- **구성**: 헤더(기록 + 🔥스트릭 배지) · 보기 토글(타임라인/월별).
- **타임라인**: 이번 주/지난주/N월 버킷 헤더, 세션 행(날짜·부위태그·제목·세트·시간·종목수), 2일+ 공백은 "휴식 N일" 표시.
  - 세션 행 탭 → **세션 미리보기 시트**(`SessionPreviewSheet`).
- **월별**: `‹ ›`로 월 이동, 날짜 그리드(운동일 강조·오늘 강조), 선택 시 하단에 그 날 세션 카드(`SessionCard`). 월 요약(횟수·총시간).
- **상태**: 기록 없으면 "아직 운동 기록이 없어요".

## 4. 종목(허브) `app/(tabs)/exercises.tsx`
- **목적**: 증권 watchlist식 종목 목록 — 1RM 추이 한눈에.
- **구성**: 헤더(종목 + 🔍) · ★보유(즐겨찾기) 섹션 · 관심(전체) 섹션.
- **요소·동작**:
  - 🔍 → 검색 입력 토글(종목명 필터).
  - 행: 종목명·부위 + 미니 스파크라인 + 현재 추정 1RM + 등락칩(▲상승/신기록=초록, ▬정체 N주=주황). **행 탭 → 종목 리포트**(`/exercise/[name]`).
  - 행 좌측 ★/☆ → 즐겨찾기 토글(=리포트 주력 핀과 공유).
  - 관심 섹션 우측 `정렬 ⌄` → 1RM순/정체순/최근순 순환.
  - 당겨서 새로고침.
- **상태**: 기록 0이면 "운동을 기록하면 종목별 추이가 여기 모여요".

## 5. 종목 리포트 `app/exercise/[name].tsx`
- **목적**: 한 종목의 진척 + AI 코치.
- **진입**: 종목 허브 행 / 정체 알림 "기록 보기" / PR 알림.
- **구성**: 헤더(‹ 뒤로 · 종목명 · ★) · 히어로(부위·종목, 큰 현재 1RM, ★PR·정체 배지, 한 줄 진단) · 차트 카드 · AI 코치.
- **요소·동작**:
  - ‹ → 뒤로, ★ → 즐겨찾기 토글.
  - **지표 토글** `[1RM][최대무게][볼륨][빈도]` → 차트 전환. 1RM·최대무게=라인(1RM엔 PR★ 마커·정체 구간 음영), 볼륨·빈도=주간 막대(이번 주 감소 강조).
  - 라인 지표일 때 **기간칩** `3M/6M/1Y/전체`.
  - 코치 3줄(진단/처방/동기). **💬 대화로 풀기** → AI 코치 대화(`stall:{종목}` 키 = 정체 알림과 같은 세션).
  - 빠른 칩 `왜 멈췄어?/리셋 방법/대체 운동` → 그 질문을 채팅에 자동 입력.
- **상태**: 로딩 스피너 / "아직 기록이 충분하지 않아요".

## 6. 운동(세션) `app/workout.tsx` — 슬라이드업 모달, 가장 복잡
- **목적**: 실시간 세션 기록.
- **시작 화면**(세션 없음): `운동 시작` 버튼, 루틴(템플릿) 목록(탭→해당 루틴으로 시작, 길게/삭제), 최근 세션 히스토리(탭→과거 세션 상세).
- **진행 중 화면**: 상단 경과 타이머 · 세션 제목(자동=부위) · 부위 태그 · 종목 카드 리스트 · `+ 운동 추가` · `완료` · `취소`.
- **종목 카드 요소·동작**:
  - **세트 행**: `무게`/`횟수`(또는 시간) 탭 → **앱 내 숫자패드**(±스텝·소수·다음 필드 이동). `✓ 완료` → 세트 저장 + 1RM 계산 + **PR 감지**(역대 최고 초과 시 햅틱) + **휴식 타이머 시작**. 완료 해제 → DB 세트 삭제·휴식 취소.
  - **세트 번호** 탭 → 타입 순환(일반→W→D→F), 길게 → 타입 직접 선택.
  - 무게 **± 버튼**(kg 2.5 / lb 5).
  - 세트 행 스와이프 → 삭제.
  - 카드 **⋯ 메뉴**: 워밍업 추가(기준무게×%·횟수), 휴식시간 설정(본세트/워밍업 다이얼), 기준 RM 변경(추정/실제·반복수), 종목 메모, 종목 삭제, 순서 변경(드래그).
- **휴식 바**(세트 사이/하단): 카운트다운, 탭 → ±조정·리셋·건너뛰기 패널.
- **완료 흐름**: `완료` → (부위 미선택 시 부위 선택 먼저) → 확인 Alert → 세션 종료 + **요약 시트**(볼륨·세트·종목·PR·시간). `취소` → 기록 삭제 확인.
- **종목 추가 모달**(다단계): 검색(전체) · ⭐즐겨찾기 · 최근 종목 빠른추가 · 부위→장비→(머신이면)브랜드→종목. 종목 **다중 선택**(체크) 후 `추가(N)`. 직접 등록(이름·측정방식), 브랜드 직접 등록.
- **과거 세션 상세**(히스토리 탭): 세트 무게/횟수 인라인 수정, 세트 추가/삭제, 날짜 변경(스피너), 제목·메모 수정, 종목 추가, `루틴으로 저장`, 세션 삭제.

## 7. Chat(허브) `app/(tabs)/chat.tsx`
- **목적**: AI 코치 대화 허브.
- **구성**: 헤더(AI 코치) · 새 소식 strip · 바로 물어보기(동적 스타터) · 최근 대화 · 새 대화 FAB(✎).
- **요소·동작**:
  - **새 소식 strip**("새 소식 N건 · 정체 N·리포트 N") 탭 → `/ai/inbox`.
  - **스타터 칩**(정체 종목 반영) 탭 → 그 질문으로 새 대화 시작·진입.
  - **최근 대화 행** 탭 → 대화 상세. 좌스와이프 → 삭제. 출처 태그(리포트/알림).
  - **FAB ✎** → 빈 새 대화 생성·진입(운동 중이면 배너 위로 올라감).
- **상태**: 대화·소식 0이면 "운동에 대해 뭐든 물어보세요".

## 8. 대화 상세 `app/chat/[conversationId].tsx`
- **목적**: AI 코치와 1:1 대화.
- **구성**: 헤더(‹ · 🤖 · 제목) · 메시지 리스트 · 입력창.
- **요소·동작**:
  - 빈 대화면 코치가 먼저 인사 + 추천 질문 칩(탭→전송).
  - 입력창 전송(↑) → 사용자 말풍선 + 타이핑 인디케이터 → **청크 스트리밍** 답변 + 후속 질문 칩.
  - 출처 맥락 칩(📊)이 있으면 상단 표시.
  - 빈 대화로 나가면 자동 삭제.

## 9. 리포트 `app/ai/reports.tsx`
- **목적**: 기간별 통합 AI 리포트.
- **구성**: 기간 단위 셀렉터(주/월/분기/반기) + 기간 버킷 칩 · 2층 리포트 뷰(`ReportView`/`ReportTabs`).
- **요소·동작**:
  - 단위·버킷 선택 → 해당 기간 리포트 로드(없으면 비동기 생성·진행률).
  - 리포트 본문: 요약 → 펼치기(상세), 종목별 진행 행 탭 → 종목 리포트, 코치 탭의 "대화 이어가기/더 물어보기" → 기간 스코프 채팅(`/ai/chat`), 주력 핀 ★·항목 숨김·어조 변경.
  - 프로필 미설정 → "먼저 목표를 알려주세요" → `설정하기`(인테이크).
  - 데이터 부족 → "완료된 운동이 있어야…" → `운동하러 가기`.
- **상태**: 로딩 중 / 불러오기 실패 → `다시 시도`.

## 10. 새 소식(인박스) `app/ai/inbox.tsx`
- **목적**: 알림 모아보기(중복 묶음).
- **요소·동작**:
  - 컴팩트 행(타입 아이콘·제목·횟수·시각, 안읽음 점). 탭 → **타입별 상세 시트**(`NotifDetailSheet`).
  - `모두 지우기` → 전체 읽음 처리.
  - 상세 시트 액션: 정체=미니 추이 + "종목 기록 보기"(→종목 리포트)·"대화로 풀기"(→채팅), PR=비교 + 기록 보기, 리포트=지표 + 리포트 열기, 운동 리마인더=운동 가기.
- **상태**: "아직 받은 소식이 없어요".

## 11. 인테이크(온보딩) `app/ai/intake.tsx`
- **목적**: 분석 렌즈 9문항(목표·경력·분할·세션시간·집중부위·통증 등).
- **요소·동작**: 채팅형 진행, 선택지 탭 또는 직접 입력, `다음`/`건너뛰기`. 마지막 `저장하고 분석 시작` → 프로필 저장 → `/ai`. 통증 등 자유 입력 칸.

## 12. 아카이브 `app/ai/archive.tsx`
- **목적**: 지난 리포트(주/월/분기…) 다시 보기.
- **요소·동작**: 시간 역순 목록, 행 탭 → 해당 기간 리포트(`/ai/reports` with type/back). 비면 "아직 지난 리포트가 없어요".

## 13. 설정 `app/(tabs)/settings.tsx` — 아코디언
- **목적**: 전역 설정(접힘 카테고리, 탭하여 펼침).
- **카테고리·동작**:
  - 👤 **계정**: 로그아웃 · 회원 탈퇴.
  - 🎯 **목표 설정**: 목표 체중 등 입력 → `저장`.
  - ⚖️ **단위**: kg/lb 토글.
  - 🎚 **표시·동작**: 세션 메모 표시·자동 부위 묻기 등 토글.
  - 🔔 **휴식 알림**: 기본 휴식 동작.
  - 📅 **운동 리마인더**: 주기(일)·시각 ± 스텝퍼(로컬 예약).
  - 💬 **AI 코치 톤**: 담백/응원/직설 칩(리포트 어조 재생성).
  - ⏱ **종목별 휴식시간**: 종목별 본세트/워밍업 휴식.
  - 🏷 **부위 관리**: 부위 태그 추가/삭제.
  - 💾 **데이터**: CSV 내보내기.
  - 🏋️ **커스텀 운동**: 추적방식(REPS/TIME) 토글, 종목 수정/삭제.
  - 🛠 **개발자 도구**: 온보딩 초기화 등.

## 14. 종목 추가/수정 `app/exercise-add.tsx` — 모달
- **목적**: 커스텀 종목 등록(부위→장비→브랜드→이름·측정방식) + 브랜드 관리.
- **요소·동작**: 단계 선택, 브랜드 선택(검색·전체·직접 등록·삭제), 운동 이름·측정방식 입력 → `저장` 또는 `저장하고 담기`(바로 세션에 추가). 기본(시스템) 종목은 이름·측정방식만 수정 가능. `담기`로 선택 종목 세션 투입.

## 개편 의뢰용 가이드

UI/UX 전반 개편을 맡길 때:

1. **개편 의뢰 프롬프트(예)**
   > "docs/SCREENS.md에 적힌 각 화면의 **기능·동작은 보존**하면서 UI/UX를 개편해줘. 화면별로 ① 현재 정보구조/인터랙션의 문제점, ② 개선안(레이아웃·위계·플로우·접근성), ③ 보존해야 할 핵심 동작을 구분해 제안하고, CARBON 다크 테마와 `constants/colors.ts` 토큰은 유지해. 큰 변경(탭 구조·내비게이션)은 먼저 합의."

2. **개편 시 보존 필수(동작 계약)** — 세트 완료→1RM/PR/휴식 자동 시작, 종목 다중선택 후 일괄 추가, 출처별 1세션 채팅(stall:/report: 키), 리포트 기간 스코프, 즐겨찾기=주력 핀 공유, 단위 kg canonical. 자세한 동작은 본문 각 화면 참조.

3. **대조 대상** — 실제 코드(`app/**`)와 이 문서가 어긋나면 코드가 진실. 동작 보존 여부는 [REQUIREMENTS.md](REQUIREMENTS.md)의 🧪(테스트된 동작)과 교차 확인.

4. **갱신** — 화면 추가/플로우 변경 시 이 문서도 함께 업데이트(살아있는 문서).
