# GymTracker UI/UX 개편 제안 (1차)

> 작성: 2026-06-18 · 기준 문서: [SCREENS.md](SCREENS.md)(현재 동작) · [REQUIREMENTS.md](REQUIREMENTS.md)(충족 기준)
> **성격**: 코드 변경 없는 설계 제안. 화면별 `문제점 → 개선안`, 그리고 **보존 동작**(깨면 안 되는 것)을 분리.
> 우선순위: **P1** 사용자 체감 큰 핵심 · **P2** 완성도 · **P3** 다듬기. 큰 구조 변경은 ⚠️로 표시(합의 필요).

## 0. 공통(cross-cutting) — 여기부터 하면 효과 큼

- **P1 · 디자인 토큰 정리**: 회색 표면값이 화면마다 제각각(`#0d0d0f`, `#161618`, `#1C1C1E`, `#1c1c22`, `#141416`…). 브랜드 레드/의미색은 토큰화돼 있으나 **surface/line/elevation 스케일이 없음**. → `SEM`에 `surface1~3·line·ink1~3`를 정의하고 전 화면을 치환. 개편의 토대.
- **P1 · 빈/로딩/에러 상태 일관화**: 화면마다 문구·레이아웃이 다름("로딩 중…", 스피너, "분석 중"). → 공통 `EmptyState`/`LoadingState`/`ErrorState` 컴포넌트(아이콘+한 줄+액션)로 통일.
- **P2 · 타이포 스케일**: 헤드라인 30/제목 20/본문 14~15가 화면마다 미세하게 다름(13.5, 14.5, 15.5…). → 5단계 텍스트 스타일로 수렴.
- **P2 · 터치 타깃 44pt**: 별★·정렬칩·세트번호 등 작은 탭 타깃이 많음. 최소 44pt 보장(히트슬롭 일관).
- **P2 · 접근성**: `accessibilityLabel`이 일부만. 아이콘 단독 버튼(톱니·🔍·★·FAB)에 라벨 전수 부여, 색만으로 구분되는 등락칩에 기호 병기(이미 ▲▬ 있음 — 유지).
- **P3 · 햅틱/사운드**: 운동 화면은 풍부, 나머지 화면은 거의 없음 → 주요 확정 액션에 가벼운 햅틱 통일.

## 1. 로그인
- **문제**: 가치 제안(왜 이 앱?)·브랜딩이 약하고 소셜 버튼 위계가 평면적.
- **개선**: 상단 로고+한 줄 카피, 1차 CTA(가장 흔한 로그인) 강조 + 나머지 보조. 약관/개인정보 링크 푸터. 첫 진입 1~2장 가치 소개(선택).

## 2. 브리핑(홈)
- **문제**: 카드 과다(헤드라인·처방·부위볼륨·지표3·시작)로 스크롤이 길고, **헤드라인·처방·부위볼륨이 모두 `/ai/reports`로 가** 목적지가 중복. 지표3 중 "체중 입력"만 동작이 달라(모달) 이질적. 생성 중 300px 고정 영역이 레이아웃 점프 유발.
- **개선(P1)**: 상단 **히어로 1개**(헤드라인+처방 통합, 탭→리포트) → 지표3 → **운동 시작(고정 하단 CTA)** 순으로 위계 단순화. 부위볼륨은 접이식/리포트로 위임. 체중은 지표가 아니라 별도 "오늘 기록" 미니 행으로 분리. 로딩은 고정 높이 스켈레톤으로 점프 제거.
- **보존**: 주간 리포트 비동기 생성·폴링·진행률, 당겨서 새로고침, 프로필 미설정 분기.

## 3. 기록
- **문제**: 월별에서 날짜 선택 시 카드가 그리드 한참 아래에 떠 **컨텍스트 점프**. 스트릭이 클라에서 재계산(백엔드 `/stats/streak`와 이원화 — N-R 정합성 위험).
- **개선(P2)**: 날짜 선택 시 그 주로 스크롤 또는 시트로 표시. 타임라인을 기본·주력으로 두고 월별은 보조 유지(현 토글 좋음). 스트릭은 백엔드 단일 소스로.
- **보존**: 타임라인 버킷/휴식갭 표기, 세션 미리보기 시트, 월 요약.

## 4. 종목(허브)
- **문제**: 등락칩 색(초록/주황) 의미를 처음엔 모름(레전드 없음). ★보유가 비었을 때/전체가 적을 때 빈 섹션 처리 모호.
- **개선(P2)**: 최초 1회 인라인 힌트("▲상승 ▬정체") 또는 헤더 작은 레전드. 보유 0이면 "관심에서 ★로 추가" 안내. 정렬 상태를 칩으로 더 명확히(현재 텍스트만).
- **보존**: watchlist 행 구성, 즐겨찾기=주력 핀 공유, 정렬/검색, 행→종목 리포트.

## 5. 종목 리포트
- **문제(P1)**: 차트에 **축 눈금·값 라벨이 없어 수치를 못 읽음**(라인/막대 공통). 탭 시 특정 지점 값도 안 보임. 기간칩이 라인 지표에만 있어 볼륨/빈도는 범위 고정.
- **개선**: y축 min/max·최근값 라벨, 포인트 탭 시 툴팁(날짜·값), 볼륨/빈도에도 기간 옵션(또는 "최근 12주" 명시 — 현재는 암묵적). 정체 음영에 "PR 이후 N주" 캡션.
- **보존**: 지표 토글, PR★·정체 음영, 코드 코치 3줄, 대화로 풀기(`stall:{종목}`), 빠른 칩.

## 6. 운동(세션) — 밀도 최우선
- **문제(P1)**: 세트 행 하나에 번호·무게·횟수·±·체크·스와이프가 모두 모여 **탭 타깃 충돌·오탭 위험**. 카드 ⋯ 메뉴에 기능 과밀(워밍업·휴식·기준RM·메모·삭제·순서). 휴식 바·숫자패드·배너가 하단에서 공간 경합.
- **개선**:
  - **P1** 세트 행 재설계: 무게/횟수는 큰 탭 영역, ±는 길게/스와이프 제스처로, 완료 체크는 우측 큰 타깃으로 분리. 스와이프 삭제는 유지하되 표준 패턴.
  - **P2** ⋯ 메뉴를 "세트 관련(워밍업·휴식·기준RM)" / "종목 관련(메모·순서·삭제)"로 그룹화한 바텀시트.
  - **P2** 하단 레이어 우선순위 규칙 문서화(숫자패드 > 휴식 바 > 배너) — 이미 일부 처리됨, 일관 규칙으로.
- **보존(동작 계약)**: 세트 완료→1RM/PR/휴식 자동 시작, 세트타입 순환(탭)·직접선택(롱프레스), 숫자패드 ±/다음, 워밍업 %계산, 종목 다중선택→일괄추가, 과거 세션 편집, 루틴 저장/시작.

## 7. Chat(허브) / 8. 대화 상세
- **문제**: 허브의 strip/스타터/최근대화 3블록 위계가 비슷해 스캔이 평면적. 대화 상세는 표준적이라 큰 이슈 없음.
- **개선(P3)**: 새 소식 strip은 있을 때만 상단 핀, 스타터는 보조 텍스트로 낮추고 최근 대화를 주력으로. 대화 상세는 출처 칩을 헤더에 통합.
- **보존**: 출처별 1세션(중복 방지), 청크 스트리밍·후속칩, 빈 대화 자동삭제, 스와이프 삭제, FAB.

## 9. 리포트 / 12. 아카이브
- **문제(P2)**: 기간 단위 셀렉터 + 버킷 칩 **2단 컨트롤**이 진입 시 무겁고, "펼치기" 깊이가 깊어 핵심 인사이트까지 탭이 많음. 아카이브와 리포트의 과거 조회 동선이 분리.
- **개선**: 기본은 "이번 기간" 한 화면으로 바로, 과거는 좌우 스와이프/드롭다운 하나로 단순화. 요약 상단에 "이번 기간 한 줄 결론 + 처방 1개" 고정, 상세는 그 아래. 아카이브를 리포트 내 기간 이동으로 흡수 검토 ⚠️.
- **보존**: 6종 기간, 2층 렌더, 기간 스코프 채팅, 주력 핀/숨김/어조, 비동기 생성·진행률.

## 10. 새 소식(인박스)
- 큰 이슈 없음(컴팩트 행 + 타입별 상세 시트 + 모두 지우기). **P3**: 타입별 색/아이콘 일관, 그룹 "N번" 표기 위치 정리. **보존**: 그룹 묶음, 상세 시트 액션 라우팅.

## 11. 인테이크(온보딩)
- **문제(P2)**: 9문항은 첫 사용 이탈 위험.
- **개선**: 진행 인디케이터(n/9), 핵심 3문항만 필수·나머지 건너뛰기 강조, 자유 입력 최소화. 중간 저장.
- **보존**: 채팅형 흐름, 프로필 저장→분석 시작.

## 13. 설정
- **문제(P2)**: 12개 아코디언 — 원하는 항목 찾기 번거로움, 검색 없음.
- **개선**: 상위 그룹화(계정 / 운동·휴식 / AI / 데이터 / 개발자) 2단, 또는 상단 검색. 개발자 도구는 릴리스 빌드에서 숨김.
- **보존**: 각 설정 동작(로그아웃·탈퇴·목표·단위·리마인더·톤·종목별 휴식·부위·내보내기·커스텀운동).

## 14. 종목 추가
- **문제(P3)**: 부위→장비→브랜드→이름 다단계가 깊음.
- **개선**: 검색 우선(이미 있음) 노출 강화, 자주 쓰는 조합 단축. **보존**: 단계 선택·브랜드 관리·"저장하고 담기".

## ⚠️ 큰 구조(내비게이션) — 합의 후 진행
- **탭바 5개 + 합성 리포트 탭**: 리포트가 실제 탭이 아니라 push라 활성 표시·뒤로가기 일관성이 약함. 후보: (a) 리포트를 브리핑 안으로 흡수해 탭 4개, (b) 리포트를 정식 탭으로 승격. → 사용자 사용빈도 데이터로 결정.
- **AI 동선 분산**: 브리핑·리포트·아카이브·채팅·인박스가 흩어짐. "AI" 허브로 묶을지 검토.

## 진행 방법(제안)
P1부터 화면 단위로: **공통 토큰/상태 컴포넌트 → 브리핑 → 종목 리포트 차트 → 운동 세트 행**. 각 화면은 SCREENS.md의 **보존 동작**을 회귀 테스트(가능하면 `npm test`/`./mvnw test`)로 지키며 개편.

> 이 문서는 1차 제안이다. 실제 개편 착수 전, 화면별로 "현 스크린샷 + 개선 시안"을 붙여 합의하는 단계를 권장.
