# GymTracker 디자인 토큰 스펙

> 최종 업데이트: 2026-06-18 · 정의 위치: `constants/colors.ts`
> **목적**: 색·타이포·형태·간격을 단일 토큰으로 통일. 화면은 [단계별로 치환](#마이그레이션). 진단 근거는 [UX_AUDIT.md](UX_AUDIT.md).
> 방향(합의됨): **정리 + 살짝 리프레시** · 의미 토큰만 확장(`SEM`) · 화면 단계별 치환.

---

## 1. 색 (`SEM`)

### 브랜드 / 상태
| 토큰 | 값 | 용도 |
|---|---|---|
| `brand` | `#FF3B30` | 액션·활성(버튼·탭·진행바·CTA). **경고 의미엔 쓰지 않음** |
| `onBrand` | `#FFFFFF` | brand 위 텍스트 |
| `danger` | `#FF453A` | 위험·삭제·파괴적 액션 + 감소 의미색 |
| `good` | `#2BD96A` | 증가·긍정·신기록 |
| `warn` | `#FFC53D` | 주의(노랑) |
| `bad` | `#FF8A00` | 미달·부족(주황) |

> **brand vs danger**: 둘 다 레드지만 의미 분리. 활성·강조=`brand`, 삭제·위험=`danger`. (기존 `#FF3B30`/`#FF453A` 혼용을 정리)

### 표면 (elevation, 어두움→밝음)
| 토큰 | 값 | 용도 |
|---|---|---|
| `bg` | `#000000` | 화면 배경 |
| `surface1` (=`surface`) | `#0D0D0D` | 기본 카드 |
| `surface2` | `#161618` | 강조 카드(히어로 등) |
| `surface3` | `#1C1C1E` | 모달·지표·세그먼트 |
| `line` | `#242427` | 기본 하헤어라인(카드 경계) |
| `line2` | `#2C2C2E` | 강한 구분선·입력 테두리 |

### 텍스트 잉크 (밝음→어두움)
| 토큰 | 값 | 용도 |
|---|---|---|
| `ink1` | `#FFFFFF` | 1차 텍스트 |
| `ink2` | `#EDEDF0` | 밝은 보조 |
| `ink3` | `#8E8E93` | 약한 보조·캡션 |
| `ink4` | `#48484A` | placeholder·disabled (**본문 금지** — 대비 부족) |

> 중간 회색(`#6a6a6e`·`#7a7a7e`·`#9a9aa2`·`#6E6E73`)은 모두 `ink3`로 수렴.

## 2. 타이포 (`TYPE`, `WEIGHT`)

| 토큰 | size / lineHeight | 용도 |
|---|---|---|
| `micro` | 10 / 14 | 칩·뱃지 |
| `caption` | 12 / 16 | 캡션·메타 |
| `footnote` | 13 / 18 | 보조 본문 |
| `body` | 15 / 21 | **본문 기준** |
| `callout` | 17 / 23 | 강조 본문·버튼 |
| `title` | 20 / 25 | 섹션 제목 |
| `headline` | 24 / 30 | 화면 제목 |
| `display` | 30 / 36 | 헤드라인 |

- **displayXl(1회성)**: 큰 히어로 숫자(현재 1RM·체중 등 40~64)는 스케일 밖에서 화면별 지정.
- **굵기** `WEIGHT`: `regular 400` · `medium 600` · `semibold 700` · `bold 800`.
- 사용 예: `{ ...TYPE.body, fontWeight: WEIGHT.semibold, color: SEM.ink1 }`.

---

## 3. 형태 / 간격 (`RADIUS`, `SPACE`)

**RADIUS(5단)**: `sm 4`(칩 내부) · `md 8`(작은 카드) · `lg 12`(카드) · `xl 16`(큰 카드·버튼) · `full 999`(알약·원형).

**SPACE(4pt 그리드)**: `xs 4` · `sm 8` · `md 12` · `lg 16` · `xl 20` · `xxl 24` · `xxxl 32`. padding·gap·margin 공통.

---

## 4. 마이그레이션 (하드코딩 → 토큰)

화면을 **하나씩** 치환한다(한 번에 X). 치환 매핑 가이드:

### 색
| 하드코딩(예) | → 토큰 |
|---|---|
| `#000` `#000000` | `SEM.bg` |
| `#0d0d0f` `#0D0D0D` | `SEM.surface1` |
| `#161618` `#1c1c22` | `SEM.surface2` |
| `#1C1C1E` `#1E1E20` | `SEM.surface3` |
| `#1A1A1A` `#15151a` `#26262b` | `SEM.line` |
| `#2C2C2E` `#2a2a30` | `SEM.line2` |
| `#FFFFFF` `#f5f5f7` | `SEM.ink1` |
| `#EDEDF0` `#e4e4ea` | `SEM.ink2` |
| `#8E8E93` `#8a8a8e` `#6a6a6e` `#7a7a7e` `#9a9aa2` `#6E6E73` | `SEM.ink3` |
| `#48484A` | `SEM.ink4` |
| `#FF3B30` (액션) | `SEM.brand` |
| `#FF453A` (삭제·위험) | `SEM.danger` |
| `#30D158` `#2BD96A` | `SEM.good` |
| `#FF9F0A` `#FF8A00` | `SEM.bad`(미달) / `SEM.warn`(노랑은 `warn`) |
| `#FFD60A` (즐겨찾기 별) | 유지(`#FFD60A`) 또는 별도 `star` 토큰 신설 |

### 타이포(스냅)
| 현재 | → |
|---|---|
| 8.5~10.5 | `micro`(10) |
| 11~12.5 | `caption`(12) |
| 13~13.5 | `footnote`(13) |
| 14~15.5 | `body`(15) |
| 16~17 | `callout`(17) |
| 18~22 | `title`(20) |
| 23~26 | `headline`(24) |
| 28~32 | `display`(30) |
| 40+ | `displayXl`(화면별 1회성) |

> ⚠️ **스냅 시 조밀 리스트 주의**: 14→15 등으로 커지면 운동·종목 같은 빽빽한 행이 줄바꿈될 수 있다. 화면별 치환 때 실제로 확인하고, 넘치면 그 화면만 한 단계 작게(footnote) 조정.

### radius / spacing
가장 가까운 토큰으로 스냅(예: 11·13→`lg 12`, 15→`xl 16`; padding 15→`SPACE.lg 16`).

## 5. 적용 현황 / 순서

- ✅ `브리핑`: 색 토큰 적용(히어로·CTA 등). 타이포/스페이싱 토큰은 후속.
- ⬜ 다음 권장 순서: 공통 컴포넌트 → 운동 → 종목/종목 리포트 → 기록 → 리포트/채팅 → 설정.
- 각 화면 치환 후 `npx tsc --noEmit` + (해당되면) `npm test`로 회귀 확인. 시각은 폰에서 점검.

## 6. 사용 규칙(요약)
- 새 코드/치환 시 **하드코딩 hex·임의 px 금지** — 토큰만 사용.
- 레드는 의미로 구분: 활성·강조=`brand`, 삭제·위험=`danger`.
- 본문에 `ink4` 쓰지 말 것(placeholder 전용).
- 토큰에 없는 값이 꼭 필요하면 먼저 토큰을 추가하고 문서를 갱신.
