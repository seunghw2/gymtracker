## 세션 시작 · 진행 유지

### (1) 진행 중 세션이 메모리에만 있어 앱 종료/백그라운드 후 사라짐
- 개선안: 진행 중 세션 상태를 기기 저장소(AsyncStorage 등)에 자동 저장 → 재시작 시 "이어서 진행" 복구.
- 단점: 서버에 이미 저장된 완료 세트와 로컬 미완료 상태의 충돌/중복 가능, 오래된 복구본이 현재와 안 맞을 수 있음, 저장 동기화 로직 복잡.

### (3) 운동 중 하단 탭바로 이탈 가능(경고 없음)
- 개선안: 다른 탭에서도 "운동 중" 플로팅 배너로 복귀 유도(또는 이탈 시 확인).
- 단점: 탭 자유도 저하, 통계·이전기록 확인 같은 정상 이동까지 거슬리게 막을 수 있음, 전역 배너 구현 복잡.

### (4) 헬스장 매번 수동 선택
- 개선안: 마지막 사용 헬스장 기본 선택(필요 시 위치 기반 추천).
- 단점: 위치 권한 요구 시 거부감, 여러 곳 다니면 틀린 기본값으로 잘못 기록 가능.

## 종목 추가 · 선택

### (6) 처음부터 이름 검색 불가
- 개선안: 종목 선택 첫 화면에 전체 검색바 상시 노출.
- 단점: 전체 종목이 많으면 결과 과다, 부위별 탐색과 UI 중복, 한/영 혼용 검색 정확도 이슈.

### (7) 운동 중 종목 연속 추가 시 전체 플로우 반복
- 개선안: 다중 선택(체크 후 한 번에 추가) 또는 추가 후 모달 유지.
- 단점: 다중선택 UI 복잡, 실수로 여러 개 추가, 각 종목 지난기록 프리필 로직과 충돌 가능.

### (8) 최근 종목 칩이 부위 단계에서만, 즐겨찾기 없음
- 개선안: 즐겨찾기(별표) + 최근 종목을 첫 화면 상단 고정.
- 단점: 즐겨찾기 저장 스키마/관리 UI 추가, 화면 복잡도↑.

### (10) 시스템 종목 측정방식 변경이 설정 깊은 곳
- 개선안: 종목 선택/카드에서 바로 측정방식 토글 접근.
- 단점: 운동 카드 복잡도 재증가(카드 토글 제거했던 결정과 상충), 실수 변경 위험.

## 세트 입력 · 숫자패드

### (11) 카드 기준 스크롤 → 아래쪽 세트가 패드에 가림
- 개선안: 행 단위 정확 스크롤(onLayout 누적 y 계산 등).
- 단점: 중첩 레이아웃에서 정확 y 계산 까다로움, 기기/폰트 크기별 편차, 스크롤 애니메이션 끊김 가능.

### (12) 숫자패드가 휴식 타이머 dock을 가림
- 개선안: 패드 표시 중 휴식 타이머를 패드 위 작은 바로 올림.
- 단점: 하단 공간 경쟁으로 화면 좁아짐, 레이아웃 복잡.

### (13) 세트 완료 후 다음 세트 연결 없음
- 개선안: 완료 시 다음 세트로 부드럽게 스크롤/강조(키패드는 자동으로 안 띄움).
- 단점: 과거 자동포커스가 키보드 띄워 불편했던 전례 → 과하면 재발, 선호 차이.

### (14) 세트 타입(W/D/F) 발견성 낮음
- 개선안: 세트 행에 타입 칩/라벨 노출, 첫 사용 코치마크.
- 단점: 행이 좁아지고 복잡, 코치마크 제작·관리 부담.

### (16) 무게 0/횟수 0으로도 완료 가능(빈 세트)
- 개선안: 0이면 완료 비활성/확인.
- 단점: 맨몸(무게 0)·시간 종목 등 정상적 0 케이스 오탐 → 예외 처리 필요.

## 종목 카드

### (19) 메모칸 2개(영구+오늘)가 항상 펼쳐져 복잡
- 개선안: 메모 기본 접힘(아이콘 탭 펼침), 내용 있으면 요약만.
- 단점: 입력에 한 번 더 탭 필요(즉시성↓), 메모 존재 인지 약화.

### (20) 순서 변경 ↑↓ 버튼만(드래그 보류)
- 개선안: reanimated + draggable-flatlist 드래그(핸들).
- 단점: 의존성 추가·Metro 재시작·메인 리스트 구조 변경, 입력/스와이프 제스처 충돌, 기기 검증 필요(보류 사유).

### (21) 슈퍼세트 중간 휴식 생략 단서 약함
- 개선안: 그룹 연결선/라벨 강조 + "→ 다음: ○○" 안내.
- 단점: 시각 요소 증가로 카드 복잡, 그룹 경계 표현 디자인 난이도.

## 휴식 타이머

### (22) 다른 탭 가면 dock 사라짐
- 개선안: 휴식 타이머를 전역 플로팅 바로.
- 단점: 전역 오버레이 구현 복잡, 타 화면 레이아웃 침범, 항상 노출 시 거슬릴 수 있음.

### (25) 무음 사운드가 포그라운드 한정
- 개선안: iOS Critical Alerts 엔티틀먼트로 백그라운드/무음에도 알림음.
- 단점: 애플 별도 승인 필요(거절 가능)·개발 빌드 필요(Expo Go 불가), 권한 요청 거부감, 남용 시 심사 이슈.

### (26) 워밍업 세트 완료에도 기본 휴식 시작
- 개선안: 워밍업은 휴식 생략 또는 짧은 휴식.
- 단점: 워밍업 휴식 선호는 사람마다 다름, 별도 설정 필요(복잡도↑).

## 과거 세션 상세 · 편집

### (27)·(28) 저장 흐름 혼란 + 추가는 즉시반영 불일치
- 개선안: 상세 편집 동작을 하나로 통일 — (a) 전부 자동저장(저장 버튼 제거) 또는 (b) 전부 명시저장 + 미저장 닫기 경고.
- 단점: (a)는 실수 수정도 즉시 반영·잦은 네트워크 호출; (b)는 깜빡하고 나가면 유실. 어느 쪽이든 트레이드오프.

### (29) 상세에서 세트 타입 변경 불가
- 개선안: 상세에서도 세트 번호 탭으로 타입 변경.
- 단점: 상세 편집 로직 복잡도↑, 저장 흐름(27/28)과 얽힘.

### (30) 과거 시간기반 세트 초 수정 불가
- 개선안: 상세에서 시간 세트 초 입력 허용.
- 단점: 상세 입력 분기 추가 + `updateWorkoutSet`에 duration 미전송이라 백엔드 PATCH 확장 필요.

## 통계

### (31) 1RM·일자 볼륨 최근 8개 고정
- 개선안: 기간 토글(1M/3M/6M/1Y/전체) + 가로 스크롤 차트, 백엔드 집계 범위 확장.
- 단점: 데이터 많으면 라벨 겹침·성능, 차트 라이브러리 한계.

### (32) PR 페이지 정렬/검색 없음
- 개선안: 부위/최근갱신 정렬 + 검색.
- 단점: 화면 복잡도↑, 정렬용 데이터(부위 등) 추가 필요.

### (33) 체중·체지방 차트 kg 고정(단위 토글 미적용)
- 개선안: 차트·체중 입력까지 단위 토글 일관 반영.
- 단점: 체중 다이얼/입력 lb 변환 일관성 필요(kg 고정했던 이유), 변환 반올림 표기 혼란.

## 캘린더

### (34) 운동한 날짜 탭해도 상세로 못 감
- 개선안: 날짜 탭 → 그날 세션 상세 모달.
- 단점: 하루 여러 세션이면 선택 UI 필요, 캘린더-운동탭 상세 모달 공유로 결합도↑.

### (35) 캘린더에서 과거 날짜 추가/진입 불가
- 개선안: 날짜 롱프레스/버튼으로 그 날짜 새 운동 시작.
- 단점: 캘린더가 입력 진입점까지 겸해 역할 비대, 실수 진입 가능.

## 홈

### (37) 당일 체중 미입력 시 진입마다 자동 팝업
- 개선안: 자동팝업 옵션화 또는 빈도 조절(주1회 등).
- 단점: 팝업 줄이면 기록 누락↑, 설정 항목 증가.

### (38) 목표 진행바가 감량 가정 → 증량 목표면 어색
- 개선안: 목표가 현재보다 높/낮음 자동 판별해 진행바·문구를 증량/감량 모드로.
- 단점: 목표≈현재일 때 모호, 문구 분기 추가, 목표 오설정 시 이상하게 보임.
