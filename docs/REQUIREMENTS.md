> **목적**: "이 앱이 무엇을 충족해야 하는가"의 기준선(baseline). 갭 점검(audit)에 사용한다 — 맨 끝 [감사 사용법](#감사-사용법) 참고. 화면별 동작 상세는 [SCREENS.md](SCREENS.md).
> **상태 범례**: ✅ 충족 · 🟡 부분 충족 · ⬜ 미구현/후속 · 🧪 자동화 테스트로 검증됨
> 이 문서는 "충족 기준"의 기준선이고, 진행 추적은 [ROADMAP](ROADMAP.md)이 담당한다(역할 분리).

제품 정의: **Strong 앱을 더 편하게 대체하는, 본인(헤비 유저)용 개인 맞춤 헬스 트래커.** 운동 기록 + "내가 잘하고 있나"에 답하는 AI 애널리스트.

## 1. 기능 요구사항 (FR)

각 항목: `[상태] 설명 — 수용 기준(검증 방법)`.

### FR-A. 인증 / 계정
- ✅ 🧪 **FR-A1** 이메일 회원가입 — 이메일·비밀번호(8자+)·이름으로 가입 시 access+refresh 토큰 발급. (`AuthFlowIntegrationTest`)
- ✅ 🧪 **FR-A2** 이메일 로그인 — 올바른 자격증명만 통과, 틀리면 4xx. (테스트됨)
- ✅ 🧪 **FR-A3** 토큰 갱신 — refresh 로테이션, 이전 refresh 재사용 거부. (테스트됨)
- ✅ **FR-A4** 소셜 로그인 — Apple / Google / Kakao OAuth. (외부 키 의존, 자동테스트 제외)
- ✅ **FR-A5** 내 정보 조회 / 로그아웃 / 회원 탈퇴(`DELETE /me`).
- 🟡 **FR-A6** 운영 인증 강제 — 현재 dev 자동인증(`dev-auth` 프로파일) 우회 경로 존재. 운영 빌드에서 게스트→로그인 강제 흐름 검증 필요.
- 수용 기준: 비로그인 상태로 보호 자원 접근 시 401. (`AuthFlowIntegrationTest`로 검증)

### FR-B. 운동 종목
- ✅ **FR-B1** 시스템 종목 시드 자동 등록(부위·장비·브랜드 메타).
- ✅ **FR-B2** 종목 목록/필터(부위·장비·브랜드) + 전체 검색.
- ✅ **FR-B3** 커스텀 종목 등록/수정/삭제, 커스텀 브랜드 추가(바텀시트).
- 수용 기준: 종목 선택은 부위→장비→브랜드→종목 4단계 + 검색으로 도달 가능.

### FR-C. 운동 세션 / 세트 (핵심)
- ✅ 🧪 **FR-C1** 세션 생성/조회/수정/삭제, 과거 날짜로 기록. (`WorkoutCrudIntegrationTest`)
- ✅ 🧪 **FR-C2** 세트 추가/수정/삭제, 세트 타입(일반/워밍업/드롭/실패) 정규화. (테스트됨)
- ✅ 🧪 **FR-C3** Epley 추정 1RM 자동 계산. (`WorkoutSetLogicTest`, `WorkoutCrudIntegrationTest`)
- ✅ **FR-C4** PR(역대 최고 1RM 경신) 감지 + 알림 적재.
- ✅ **FR-C5** 이전 세션 값 자동 프리필, 워밍업 세트 자동 생성. 🧪(`exerciseEntry.test`)
- ✅ **FR-C6** 스트롱식 인라인 휴식 타이머(세트 사이 라인, 카운트다운, ±조정, 사운드/로컬알림, 자동종료).
- ✅ **FR-C7** 진행 중 운동 전역 배너(다른 탭에서 표시·복귀).
- ✅ **FR-C8** 세션/종목 메모 입력.
- 🟡 **FR-C9** 세션 시작 시 헬스장 선택 — 백엔드 gymId는 있으나 UI 미연결(보류, 우선순위 낮음으로 판단됨).

### FR-D. 신체 기록
- ✅ **FR-D1** 체중 날짜기준 upsert + 최신 조회, 홈 다이얼 입력.
- 🟡 **FR-D2** 체지방률 — 백엔드 지원되나 입력 UI·추이 차트·목표 연동 미완.

### FR-E. 통계 / 종목별 리포트
- ✅ **FR-E1** 1RM 성장 차트(추정/실제 N-RM 전환), 볼륨 추이, 부위별 빈도, 기간 요약.
- ✅ 🧪 **FR-E2** 종목 요약 일괄 조회(`/stats/exercise-summary`) — 전 종목 현재/PR 1RM·정체주수·trend·스파크. (`ExerciseStatsServiceTest`, `StatsExerciseIntegrationTest`)
- ✅ 🧪 **FR-E3** 종목 진척 시계열(`/stats/exercise-progress`) — 1RM·최대무게 일별, 볼륨·빈도 주별. (테스트됨)
- ✅ **FR-E4** 종목 탭 watchlist 허브 — ★보유/관심, 미니 스파크, 등락칩, 정렬(1RM/정체/최근)·검색.
- ✅ **FR-E5** 종목 리포트 화면 — 큰 숫자 헤더, 지표 토글 차트(1RM은 PR★·정체 음영), 코드 템플릿 코치, "대화로 풀기".
- ✅ **FR-E6** CSV 내보내기.

### FR-F. AI 리포트 (애널리스트)
- ✅ **FR-F1** 온보딩 인테이크(9문항)로 분석 렌즈 설정.
- ✅ **FR-F2** 기간 리포트 6종(세션/주/월/분기/반기/연) 2층 렌더 + 기간 셀렉터.
- ✅ 🧪 **FR-F3** "계산은 코드, 해석만 LLM" — 수치는 백엔드 확정, LLM 출력은 검증기로 환각 차단·반올림 허용. (`AiReportValidatorTest`)
- ✅ 🧪 **FR-F4** 정체(stagnation) 탐지 — N주 신기록 없음, asOf 기준. (`ReportAggregationServiceTest`)
- ✅ **FR-F5** 비동기 생성 + 실시간 진행률(스트리밍 토큰 비례) + 재조회 시 즉시 표시(stale-while-revalidate).
- ✅ **FR-F6** 어조 설정(담백/응원/직설) 반영, 아카이브(과거 리포트).
- 🟡 **FR-F7** 남은 시각화 — 목표 진척 게이지/요일 히트맵/체중 추세/일관성 score/타임라인 세션화(후속).

### FR-G. AI 코치 채팅
- ✅ 🧪 **FR-G1** 세션 기반 대화 — 출처별(stall:{종목}/report:{기간}) 1세션 중복 방지(findOrCreate 멱등), 유저 격리. (`ChatConversationIntegrationTest`)
- ✅ **FR-G2** 허브(알림 strip·동적 스타터·최근 대화·새 대화 FAB) + 상세(청크 스트리밍·타이핑·후속칩).
- ✅ 🧪 **FR-G3** 빈 대화 자동 삭제, 스와이프 삭제. (`useChatStore.test` 리듀서 일부)
- ✅ **FR-G4** 기간 스코프 채팅 — 보는 리포트 데이터에 한정, 스코프 밖 질문 거절.

### FR-H. 알림
- ✅ 🧪 **FR-H1** 표시 그룹핑(중복 묶음·횟수·최근시각·안읽음). (`groupNotifications.test`)
- ✅ **FR-H2** 인박스 화면 + 타입별 상세 시트(정체/PR/리포트), 안읽음 배지·전체 읽음.
- ✅ **FR-H3** 적재: 리포트완료·PR·정체·리마인더, `(user_id, dedupe_key)` 중복 방지.
- 🟡 **FR-H4** 진짜 원격 푸시 — Expo Go 한계로 미구현. 운동 리마인더=로컬 예약, 리포트완료=인앱 폴링으로 대체. 실제 푸시는 EAS dev build 후속(푸시 토큰 엔드포인트는 존재).

### FR-I. 설정 / 기타
- ✅ **FR-I1** 유저별 key-value 설정(아코디언 UI).
- 🟡 **FR-I2** kg/lb 단위 토글 — 변환 유틸은 있으나(🧪 `units.test`) 전 화면 일관 적용 검증 필요.
- ✅ **FR-I3** 종목별 기준 RM(추정/실제·반복수), 종목별 휴식, 주력 핀(=종목 즐겨찾기).
- ✅ **FR-I4** Strong CSV 임포트(세션/세트/종목 매핑, 워밍업·드롭·실패 보존).
- 🟡 **FR-I5** 루틴 템플릿 — 조회/삭제 엔드포인트만 존재(생성·적용 흐름 불완전, 후속 확인 필요).

## 2. 비기능 요구사항 (NFR)

> 측정 가능 목표 위주. 갭 점검의 핵심 대상.

### NFR-성능
- 🟡 **N-P1** 리포트 재조회 즉시성 — 한 번 본 리포트는 네트워크 대기 없이 즉시 표시(persist + 폴백). *목표: 체감 <300ms.* (구현됨, 수치 측정 미실시)
- ✅ **N-P2** 종목 허브 — 전 종목 요약을 호출 1회로(N+1 회피). (`exercise-summary` 단일 쿼리 그룹핑)
- ⬜ **N-P3** 앱 콜드 스타트/리스트 스크롤 성능 목표치 미정의(측정 필요).

### NFR-보안
- ✅ 🧪 **N-S1** 유저 데이터 격리 — 모든 쿼리 userId 스코프, 타 유저 자원 접근 불가. (`ChatConversationIntegrationTest` 격리 검증)
- ✅ 🧪 **N-S2** JWT 인증(access 60분/refresh 14일) + refresh 로테이션. (테스트됨)
- ✅ **N-S3** 비밀번호 BCrypt, API 키·시크릿 하드코딩 금지(env). dev 자동인증은 `dev-auth` 프로파일로 격리.
- 🟡 **N-S4** 운영 시 dev 우회 비활성 보장 — 프로파일 분리 최종 점검 필요(FR-A6 연동).

### NFR-신뢰성 / 정합성
- ✅ **N-R1** LLM 환각 방지 — 수치는 코드 확정, 출력 검증 실패 시 1회 재시도→직전 성공 폴백(같은 기간). 🧪
- ✅ **N-R2** 무게 canonical = kg(표시/입력만 변환), 1RM Epley 단일 정의.
- ✅ **N-R3** 기간 스코프 정합성 — 과거 기간 리포트에 미래 기록 누수 없음. 🧪(`ReportAggregationServiceTest` asOf)
- ✅ **N-R4** 네트워크 실패에 화면 안 죽음(스토어 try/catch, 빈 상태 처리).
- 🟡 **N-R5** DB 마이그레이션 — 현재 `ddl-auto=update`(컬럼 자동추가, 삭제 안 됨). 운영은 Flyway + `validate` 전환 권장(미도입).

### NFR-품질 / 유지보수
- ✅ 🧪 **N-Q1** 자동화 테스트 — 백 43 + 프론트 35 = 78. 핵심 도메인 커버. (`./mvnw test`, `npm test`)
- ✅ **N-Q2** 프론트 타입 안전(`tsc --noEmit` 통과), 디자인 토큰 단일 출처(`constants/colors.ts`).
- 🟡 **N-Q3** 커버리지 목표치 미설정(현재는 핵심 로직 위주, 컴포넌트 렌더 테스트 없음).

### NFR-UX / 접근성
- ✅ **N-U1** 다크 전용 CARBON 테마, 의미색(증가 그린·경고 오렌지)과 브랜드 레드 분리.
- ✅ **N-U2** 전 화면 한국어, 종목명 등 원문 유지.
- 🟡 **N-U3** 접근성(스크린리더 라벨·대비비·터치 타깃 44pt) — 부분 적용(`accessibilityLabel` 일부), 전수 점검 미실시.

### NFR-운영 / 이식성
- ✅ **N-O1** 백엔드 고정 도메인(cloudflared named tunnel → `api.hammerslog.trade`).
- 🟡 **N-O2** 배포 — 백엔드 클라우드 정식 배포 미전환(로컬+터널 운영). 프론트 EAS 빌드 미전환(Expo Go).
- ✅ **N-O3** 확장 대비 — `report_type`·기간 파라미터 일반화, `LlmClient` 인터페이스(Anthropic/Bedrock 골격).

## 감사 사용법

이 문서를 기준으로 앱의 미흡한 점을 점검할 때:

1. **갭 스캔 프롬프트(예)**
   > "docs/REQUIREMENTS.md를 기준선으로, 실제 코드(백엔드 컨트롤러/서비스, 프론트 화면/스토어)와 대조해 ① 🟡·⬜ 항목 중 사용자 영향이 큰 순서, ② ✅로 적혔지만 실제 구현이 빈약하거나 엣지케이스가 빠진 곳, ③ 문서에 없는데 코드엔 있는 기능을 찾아줘. 추측 말고 파일 인용으로."

2. **대조 대상** — 백엔드 `*Controller.java`(엔드포인트 실재), 프론트 `app/**`(화면), `src/test`·`lib/__tests__`(🧪 표시가 실제 테스트와 일치하는지), [ROADMAP](ROADMAP.md)(진행 중/보류 사유).

3. **신뢰 규칙** — 🧪 표시 항목만 "검증됨"으로 신뢰하고, 나머지는 "구현됐다고 주장됨"으로 간주해 직접 코드/동작 확인. 상태(✅/🟡/⬜)가 코드와 어긋나면 그 자체가 발견 사항.

4. **갱신** — 기능 추가/변경 시 해당 FR/NFR의 상태와 🧪 표시를 함께 업데이트(살아있는 문서).
