# GymTracker · AI 범위 감사 (무엇을 AI / 무엇을 숫자)

> 원칙: **숫자는 "사실"을, AI는 "판단"을.** 계산되는 건 숫자로(빠르고 정확), AI는 *고르고·해석하고·대화하는* 것만.
> 자가 테스트(출력마다): ① "숫자만 봐도 아나?"→YES면 숫자로 ② "보고 내가 뭘 바꾸나?"→NO면 빼라.

## 현 AI 사용 감사
(LLM 출력 = `AiReportOutput`; 나머지는 백엔드가 규칙으로 계산. 검증기가 LLM 수치를 실데이터와 대조해 환각 차단.)

| 출력 | 생성 | 판정 | 이유 / 조치 |
|---|---|---|---|
| **헤드라인 한 줄** ("어깨·등 집중, 잘 밀어붙였어") | LLM | ✅ **유지** | 숫자 10개→사람 말 1줄. 프롬프트가 이미 "원시 숫자 나열 금지" |
| **처방 1개** (action·why) | LLM | ✅ **유지** | 지표 20개 중 *제일 중요한 행동 하나* — 가장 가치 큼 |
| **코치 탭 / 채팅 Q&A** | LLM | ✅ **유지**(점검) | 내 데이터 기반 대화. 단 숫자 재진술 항목은 솎기 |
| `summaryMetrics` (LLM이 고른 지표) | LLM·**레거시** | 🔻 **제거** | 코드 KPI가 이미 주력(DTO에 "레거시·선택"). LLM 버전 불필요 |
| `strengths` (강점 리스트) | LLM·**레거시** | 🔻 **제거** | 숫자 재진술. 코드 하이라이트(▲ PR)가 커버 |
| `watchouts` (주의 리스트) | LLM·**레거시** | 🔻 **제거** | 정체·부위 부족은 코드가 계산(주황 표시) |
| 데이터 탭 카드(볼륨·균형·일관성·강도…) | 코드 | 📊 **숫자 유지** | 이미 계산. 색 임계값(좋음/주의)으로만 |
| LBM/목표 코멘트 ("지방만 감소") | 코드(규칙 if/else) | 📊 **유지** | 결정적·근거 있음(LLM 아님). 환각 없음 |
| 마일스톤·하이라이트·정체/PR | 코드 | 📊 **유지** | 계산값 |

## 결론
- 앱은 **이미 ~90% 맞음** — "계산은 코드, 해석만 LLM" 원칙이 지켜지고 있고, 군더더기 LLM 필드는 이미 *레거시*로 표시됨.
- **남은 일 = 레거시 LLM 필드 3개(`summaryMetrics`·`strengths`·`watchouts`) 제거 + 코치 탭에서 "숫자 재진술" 문장 솎기.**
- 결과: AI 표면이 **헤드라인 · 처방 · 채팅** 3개로 좁혀져 → 더 빠르고·덜 틀리고·더 신뢰.

## 효과 검증(타겟=본인)
- "헤드라인/처방이 (a) 내가 몰랐던 걸 알려주나 (b) 운동을 바꾸게 하나?" 둘 다 NO면 숫자로 강등.
- 지표: 처방 실행률 · 브리핑 재방문 · 채팅 사용률.

## 액션 (별도 작업으로)
1. 백엔드 `AiReportOutput`에서 `summaryMetrics`/`strengths`/`watchouts` 제거 + 프롬프트에서 해당 지시 삭제(토큰·지연↓).
2. 코치 탭 챕터 중 숫자 재진술 항목 점검.
3. 프론트에서 이 필드 참조 잔재 정리.
